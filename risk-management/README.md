# リスク管理フレームワーク

**「どれだけ利益を狙うか」ではなく「どれだけ損失を許容するか」から始める**
リスク管理の、9部構成の実装＋実データデモ。成熟した運用組織が重視する
「想定外の損失を早く検知し、リスクを定量的に可視化し、ルールを一貫適用する」
という運用プロセスを、手を動かして学ぶ。

> 研究・学習用。発注機能は持たない。実運用は別の正規手段（例: 楽天証券RSS）で。

## 9部構成と実装の対応

| # | テーマ | 実装 | 主な機能 |
|---|---|---|---|
| 1 | **リスク予算** | `limits.py` | 階層別の上限定義・制約チェック・VaR（パラメトリック/ヒストリカル） |
| 2 | **ポジションサイズ** | `sizing.py` | 逆ボラ・目標ボラ・リスクパリティ・リスク寄与分解 |
| 3 | **損失管理** | `stops.py` | 価格/ATR/時間/シグナル消失の4種ストップを重ねる |
| 4 | **ドローダウン管理** | `drawdown.py` | 警戒→縮小→停止の段階的対応 |
| 5 | **相関監視** | `market_risk.py` | ローリング平均相関・急上昇検知・ストレス時相関 |
| 6 | **ストレステスト** | `market_risk.py` | 一律ショック・ボラ急騰シナリオでの損益 |
| 7 | **流動性評価** | `liquidity.py` | 平均売買代金・想定売却日数 |
| 8 | **毎朝チェックリスト** | `daily_report.py` | 上記を1枚に束ねた運用前レポート |
| 9 | **運用思想** | （全体） | 利益の最大化より、損失検知・可視化・一貫性を重視 |

## セットアップ & 実行

```bash
cd risk-management
pip install -r requirements.txt
python demo.py    # 分散6銘柄ポートで全機能を通し実演
```

## 各部の要点

### 1. リスク予算（`limits.py`）
ポジション / 戦略 / ポートフォリオの階層で損失上限を先に決める（`RiskLimits`）。
`check()` が現状値と上限を突き合わせ、違反を洗い出す。VaRは正規近似（パラメトリック）と
分布の裾を直接使うヒストリカルの両方を用意（裾リスクは後者の方が正直）。

### 2. ポジションサイズ（`sizing.py`）
- **逆ボラ**: 低ボラ銘柄を大きく（簡便）
- **リスクパリティ**: 各銘柄の *リスク寄与* を均等化（`risk_contribution` の pct が揃う）
- **目標ボラ**: ポート全体を目標年率ボラに合わせてスケール → レバor現金比率が決まる
- デモ実測: リスクパリティで寄与率が全銘柄16.7%に揃い、分散が効いていることを確認

### 3. 損失管理（`stops.py`）
価格%・ATR（ボラベース）・時間・シグナル消失の4種を重ね、**最初に発火したもの**で
手仕舞う。一つのルールへの依存を減らす。

### 4. ドローダウン管理（`drawdown.py`）
累積損失が閾値を超えたら 警戒（監視強化）→ 縮小（エクスポージャー削減）→
停止（新規停止＋戦略レビュー）と段階的に対応。`exposure_scale` を返し、
サイジングに掛けて自動縮小できる。

### 5. 相関監視（`market_risk.py`）
平常時は分散されていても、ストレス時に相関が跳ね上がり分散が効かなくなる。
デモ実測: **平常時 平均相関 0.29 → 市場下落日 0.51（+0.22）**。
「必要なときに限って分散が消える」を数字で確認できる。

### 6. ストレステスト（`market_risk.py`）
一律ショック（-10%/-20%）やボラ急騰（2σ/3σの1日下落）でポート損益を評価。
過去の暴落を思い出さなくても、耐性の当たりをつけられる。

### 7. 流動性評価（`liquidity.py`）
`想定売却日数 = 建玉 ÷ (参加率 × 平均売買代金)`。1日で抜けられない建玉に印をつける。
価格が良くても現実的に手仕舞えなければ意味がない。

### 8. 毎朝チェックリスト（`daily_report.py`）
損益 / エクスポージャー / ボラ / 相関 / 流動性 / VaR・ストレス / DD対応 / 制約違反を
1枚に。デモでは単一ポジション上限やボラ上限の違反を自動で ⚠️ 表示する。

## 限界・注意
- 教育用の簡易実装。実務の厳密さ（時価評価の細目・約定モデル・税・借株）は持たない
- VaR・ストレスは正規近似や単純シナリオ。テールリスクは過小評価されがち
- 相関・ボラは過去推定であり、将来のレジーム変化には遅れて反応する
- 発注機能なし。**このフレームワークは「測る・止める」ためのもの**で、
  儲けるロジックは別（`../aqm-strategy/` 等）。両者を分けて設計するのが健全
