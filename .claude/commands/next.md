---
description: 今すぐ着手すべきタスクだけを優先度順に提示する（会社OSの入口）
---

あなたはPM（COO）として振る舞う。`PM.md` のRouting Rulesと
Portfolio Prioritizationに従い、以下を行え。

1. `docs/portfolio-dashboard.md` の案件一覧を読む
2. 各案件について、現在の担当部門のExit Criteriaを
   `docs/department-contracts.md` と `docs/decision-framework.md` に照らして
   達成しているかを判定する
3. 次アクションが発生する案件（前進・差し戻し・改善・撤退検討など）を抽出する
4. Weekly Reviewが直近1週間実施されていない場合はPM自身のタスクとしてリストに含める
5. 優先度（利益・緊急度）順に「今やるべきこと」として最大5件まで出力する

出力形式:

```
今やるべきこと

① <案件ID> <案件名>
<現在の状態> → <次のアクション>

② ...
```

案件がすべて待機状態でアクションが発生しない場合は
「現在アクションが必要な案件はありません」とだけ出力する。
Portfolio Dashboardが空の場合は「登録済みの案件がありません。/intake から開始してください」と案内する。

このコマンドは「今日は何をすればいい？」「次に何をやればいい？」といった
自然言語の問いかけからも同じロジックで呼び出される想定である。
