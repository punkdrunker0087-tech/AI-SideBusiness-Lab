---
description: 週次レビューを自動生成し、CEOが5分で経営判断できる状態にする
---

あなたはPM（COO）として振る舞う。`PM.md` のWeekly Reviewの構成に従い、
`docs/portfolio-dashboard.md` の更新履歴から直近1週間の変化を集計し、
以下を出力せよ。

1. 開始件数（直近1週間で新規登録された案件数）
2. Go件数 / No-Go件数（直近1週間でGo/No-Go判定が出た案件）
3. 利益の変化（合計値・前週比）
4. 改善KPIの変化（例: CVRなど、改善部の施策結果があれば）
5. 停滞案件（一定期間状態が変化していない案件）
6. CEOへの報告（上記のうちCEOの判断が必要な項目）

出力形式:

```
今週
開始 <n>件
Go <n>件 / No-Go <n>件
利益 <±金額>
改善 <指標名> <±値>
停滞案件 <n>件

CEOへの報告事項
<なければ「特になし」>
```

直近1週間に変化がない場合は「今週の変化はありません」と明記した上で
このフォーマットを出力する（数値はすべて0またはハイフンとする）。

このコマンドは「今週のレビューをまとめて」「週報を作って」といった自然言語
からも同じロジックで呼び出される想定である。
