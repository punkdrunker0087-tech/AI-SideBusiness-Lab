---
description: 今日のX投稿・note記事案を作成する（マーケティング部・日次コンテンツ）
argument-hint: [案件ID（省略可、デフォルトPJ-002）] [トピック（省略可）]
---

あなたはマーケティング部として振る舞う。対象: $ARGUMENTS
（案件ID・トピックの指定がなければPJ-002を対象とする）

これは`/marketing`（新商品の6点セット作成）とは別の、既存商品の日次
SNS運用コマンドである。`Marketing.md`の6点セット・Exit Criteria手続きは
呼び出さない。

1. 実際の日付を確認する（架空の日付・実績を作らない）
2. `docs/daily-metrics.md`の最新報告済み実績を確認する。未報告の項目は
   「未報告」として扱い、数字を憶測で埋めない
3. `docs/channel-strategy.md`の投稿頻度ルール（1日3本・可能な限り
   画像付き）と80:20原則・書き出しルールに従い、**X投稿を3本**作成する
   （開発ログ・気づき・経過中心。商品紹介は1本まで、ブランド名は文末のみ）。
   **各投稿に「画像指示」を必ず添える。** 画像指示は以下の2形式の
   どちらかで、人間が迷わず用意できる粒度まで具体化する:
   - **撮影型**: スクリーンショットの取得対象を指定（どのツールの
     どの画面・どんなデータを入れた状態か・PC/スマホどちらの表示か。
     例:「売上台帳のダッシュボードタブ。テストデータで棒グラフが
     6ヶ月分見える状態。PC表示」）
   - **作成型**: 図解・カード画像の作成指示（構図・入れる文字・配色は
     ブランドカラー紺#101a2b×黄#ffd333・サイズ。サムネイル指示書と
     同じ粒度。例:「Before/After比較カード。左『手作業3時間』右
     『タップだけ』、中央に矢印、下部に商品名」）
   どちらの場合もAnthropicロゴ等の商標は使わず、数字を載せる場合は
   実績値のみ（架空の数字を画像に入れない）。
   **各投稿に「推奨投稿時刻」を必ず添える。** `docs/channel-strategy.md`
   の投稿時間帯ルールに従い、3本を朝・昼・夜のゴールデンタイムに割り
   当てる。目安（平日）: ①朝7〜9時（通勤層・経過/気づき系）②昼12〜13時
   （昼休み）③夜20〜22時（最大リーチのゴールデンタイム。最も届けたい
   1本＝商品紹介や買い手向けをここに）。土日は13〜24時（特に18〜22時）に
   寄せる。初動6時間が勝負なので遅すぎる時刻を避ける
4. note記事は毎日ではなく週1本を目安とする（`Marketing.md`のnote投稿
   達成率の基準と同じ）。直近7日以内にnote記事を作成済みの場合は省略し、
   その旨を明記する
5. 出力はその場の下書きとする。ファイルへの保存・Exit Criteriaの判定・
   Portfolio Dashboardの更新は行わない。CEOが気に入った投稿がある場合の
   み、既存ファイル（`cases/<ID>/marketing*.md`等）への反映を提案する
6. 実際の投稿・出品操作は人間が行う（`Marketing.md`「自動化の境界」）

出力形式:

```
X投稿案:
1. [推奨: 平日07-09時] <本文> ／ 画像指示: <撮影型/作成型>
2. [推奨: 平日12-13時] <本文> ／ 画像指示: <撮影型/作成型>
3. [推奨: 平日20-22時] <本文> ／ 画像指示: <撮影型/作成型>
note記事案: <作成 / 週内作成済みのため省略>
```

このコマンドは「今日のSNS投稿を考えて」「Xとnoteの下書きちょうだい」
といった自然言語からも同じロジックで呼び出される想定である。
