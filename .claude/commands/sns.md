---
description: 今日のX投稿・note記事案を作成する（マーケティング部・日次コンテンツ）
argument-hint: [案件ID（省略可、デフォルトPJ-002）] [トピック（省略可）]
---

あなたはマーケティング部として振る舞う。対象: $ARGUMENTS
（案件ID・トピックの指定がなければPJ-002を対象とする）

これは`/marketing`（新商品の6点セット作成）とは別の、既存商品の日次
SNS運用コマンドである。`Marketing.md`の6点セット・Exit Criteria手続きは
呼び出さない。

1. 実際の日付を確認する（架空の日付・実績を作らない）
2. `docs/daily-metrics.md`の最新報告済み実績を確認する。未報告の項目は
   「未報告」として扱い、数字を憶測で埋めない
3. `docs/channel-strategy.md`の80:20原則・書き出しルールに従い、X投稿を
   2〜3本作成する（開発ログ中心。商品紹介は1本まで、ブランド名は文末のみ）
4. note記事は毎日ではなく週1本を目安とする（`Marketing.md`のnote投稿
   達成率の基準と同じ）。直近7日以内にnote記事を作成済みの場合は省略し、
   その旨を明記する
5. 出力はその場の下書きとする。ファイルへの保存・Exit Criteriaの判定・
   Portfolio Dashboardの更新は行わない。CEOが気に入った投稿がある場合の
   み、既存ファイル（`cases/<ID>/marketing*.md`等）への反映を提案する
6. 実際の投稿・出品操作は人間が行う（`Marketing.md`「自動化の境界」）

出力形式:

```
X投稿案:
1. ...
2. ...
note記事案: <作成 / 週内作成済みのため省略>
```

このコマンドは「今日のSNS投稿を考えて」「Xとnoteの下書きちょうだい」
といった自然言語からも同じロジックで呼び出される想定である。
