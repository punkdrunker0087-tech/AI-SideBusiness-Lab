# 日本株バックテスト基盤（楽天証券想定）

日本株・ETFの売買戦略を**過去データで検証**するための基盤。
「戦略を作る前に、そもそもエッジがあるか確かめる」という一連の方針の到達点。

- **検証（このツール）**: 無料の過去データ（Yahoo Finance）でバックテスト
- **実行（別途・あなたのPC）**: 楽天証券のマーケットスピードII RSSで発注

## なぜこの構成か

Polymarketでの実取引は日本居住者にとって法的リスクがあり断念、その確率を
シグナルに使う検証でも予測力は確認できなかった（`../polymarket-signal-research/`）。
そこで、**合法性が明快で、自前データでエッジ検証まで完結できる日本株**に軸足を
移した。楽天証券の口座で使えるRSSが実行手段になる。

## 設計思想：過学習を疑う

このツールの主役は派手な戦略ではなく、**「その戦略は本当に効くのか」を
厳しく問う仕組み**である。

- **先読み防止**: close[t]で決めたポジションは翌日リターンに適用（1日ずらす）
- **取引コスト**: ポジション変更時にbpsで課金（既定5bps。楽天の国内株手数料は
  ゼロコースなら0だが、スプレッド/スリッページを保守的に見込む）
- **イン/アウト分割**: 期間をtrain/testに分け、**アウトオブサンプルでバイ&ホールドを
  上回るか**を最重要判定にする。イン側だけ良い戦略は「過去に当てはめただけ」
- **多重比較の警告**: 銘柄・戦略・パラメータを多く試すほど「偶然の勝ち」が混じる。
  最終判断前に別銘柄・別期間での再現を必須とする

## 構成

| ファイル | 内容 |
|---|---|
| `data.py` | Yahoo Finance経由で日本株/ETF/指数の日足を取得（CSVキャッシュ） |
| `strategies.py` | 売買シグナル生成（SMAクロス・モメンタム・RSI逆張り・B&H） |
| `backtest.py` | バックテストエンジンと成績指標（Sharpe・最大DD・勝率等） |
| `run.py` | 検証CLI（train/test分割・B&H比較・判定） |

## セットアップ

```bash
cd rakuten-backtest
pip install -r requirements.txt
```

## 使い方

```bash
# 日経225連動ETFで全戦略を一覧比較
python run.py --symbol 1321.T --strategy all

# トヨタでSMAクロス（パラメータ指定）
python run.py --symbol 7203.T --strategy sma_crossover --fast 25 --slow 75

# モメンタム戦略・取引コストを厳しめに
python run.py --symbol 1321.T --strategy momentum --lookback 60 --cost-bps 10
```

シンボル例: `1321.T`（日経225ETF）, `1570.T`（日経レバETF）, `7203.T`（トヨタ）,
`6758.T`（ソニーG）, `9984.T`（ソフトバンクG）, `^N225`（日経225指数）

## 結果の読み方

```
アウト(test) : 総リターン +12.0%  CAGR +15.7%  Sharpe +0.69  最大DD -19.9% ...
└ 基準:B&H   : 総リターン  +4.2%  CAGR  +5.5%  Sharpe +0.33  最大DD -31.9% ...
判定: ○ アウトでB&Hを上回る  [超過リターン +7.7%pt / 超過Sharpe +0.36]
```

- **見るべきは「アウト(test)」の行と、その下の基準(B&H)との差**だけ
- イン(train)がどれだけ良くても、アウトでB&Hに負けるなら実運用価値はない
- `○`が出ても鵜呑みにしない。**別銘柄・別期間で再現するか**を必ず確認する
  （1つの`○`は多重比較の偶然でありうる）

## 楽天証券RSSへの橋渡し（実行フェーズ）

このバックテストで**アウトオブサンプルでも安定してB&Hを上回る戦略**が
見つかった場合に限り、実行を検討する。実行はあなたのWindows PC上で行う:

1. 楽天証券でマーケットスピードIIをインストールし、RSS利用を申請
   （総合口座があれば無料。要「確認書兼同意書」への同意）
2. Excelで、RSS関数 `=RssMarket(...)` 等でリアルタイム株価を取得
3. `strategies.py` と同じロジックをExcel VBA（またはPython+xlwings）で再現し、
   シグナルが出たら `=RssOrder(...)` 相当で発注
4. **最初は必ず少額・低頻度で。** バックテストの成績は将来を保証しない

> 注意: RSSはWindows専用・Office 2013/2016/2019対応（365非対応）。
> 詳細は `../docs/articles/` および楽天証券公式のRSS仕様を参照。

## 限界・注意（必ず読む）

- **バックテストの好成績は将来の利益を保証しない。** 過去に効いた ≠ これから効く
- **Yahoo Financeの日足は無配当調整・分割調整の扱いに癖がある。** 厳密な検証には
  配当込みトータルリターンや、楽天証券自身のデータでの再検証が望ましい
- **日足・終値ベースの簡易モデル。** 実際の約定はザラ場で動き、スリッページ・
  板の薄さ・寄り引けの差が成績を削る。`--cost-bps` を厳しめにして耐性を見ること
- **強気相場ではB&Hが非常に強い。** 「B&Hに勝てない」は失敗ではなく、
  「余計なことをせず持ち続けるのが最適」という立派な結論でもある
- このツールは**検証のみ**で発注機能を持たない。実発注は楽天証券RSSで別途行う
