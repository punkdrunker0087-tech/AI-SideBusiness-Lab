# Polymarket シグナル研究（読み取り専用）

Polymarketを**賭け場としてではなく、確率の計測器として**使い、その確率が
日本で合法的に取引できる金融商品（ドル円・日経225など）の値動きに対して
**予測力（先行性）を持つか**を実データで検証するツール。

## なぜこの形なのか

`polymarket-trading-bot/` で検討したPolymarketでの実取引（賭け）は、
日本居住者にとって賭博罪該当や口座凍結のリスクが判明したため断念した。
一方で、Polymarketの**公開されている価格＝確率を「読む」だけ**なら賭けでは
なく公開情報の分析であり、そのリスクの外にある。

そこで戦略を次のように組み替えた:

1. **Polymarket（読み取り専用）** — 特定イベントの確率を時系列で取得
2. **合法的な金融商品** — 楽天証券等で取引できる商品（ドル円・日経）の価格を取得
3. **先行性を検証** — 「Polymarketの確率変化は、金融商品の値動きに先行するか？」
4. エッジが確認できて初めて、合法な商品側でのドライラン検証に進む

この形なら**実資金もAPIキーもウォレットも不要**で、賭博リスクゼロのまま
「Polymarketデータに予測力があるか」を確かめられる。

## 何を検証しているか（重要）

Polymarketの価格はすでに群衆が織り込んだ「効率的」な確率であることが多い。
つまり、あなたがそれを見た時には金融市場も同じ情報を織り込んでいる可能性が
高く、**それ単体では優位性（エッジ）にならない**。エッジが生まれるのは、
Polymarketの反応が金融商品の反応より**速い（先行する）**歪みがある時だけ。

このツールは、まさにその「先行しているか？」をラグ別の相互相関で測る。

## 構成

| ファイル | 内容 |
|---|---|
| `polymarket_source.py` | Polymarketの市場一覧・全文検索・価格履歴の取得（読み取り専用） |
| `market_source.py` | Yahoo Finance経由で日本の金融商品の価格を取得 |
| `analyze.py` | 2系列を整列し、ラグ別相互相関＋並べ替え検定で分析 |
| `research.py` | 単一ペアを詳しく見るCLI |
| `scan.py` | 複数ペアを総当たり検証し、偽陽性を補正して結論を出すCLI |

## セットアップ

```bash
cd polymarket-signal-research
pip install -r requirements.txt
```

## 使い方

```bash
# 対象になりそうな市場を探す（例: Fed金利）
python research.py --list --keyword Fed

# 出来高トップのFed市場 と ドル円 の先行性を分析
python research.py --keyword Fed --symbol USDJPY=X --save data/fed_usdjpy.csv

# 日経225と比べる / 別の市場を選ぶ
python research.py --keyword "decrease interest rates by 25" --symbol ^N225

# YESトークンIDを直接指定
python research.py --token <yes_token_id> --symbol USDJPY=X
```

主なシンボル: `USDJPY=X`（ドル円）, `^N225`（日経225）, `^TPX`（TOPIX）,
`1570.T`（日経レバETF・楽天証券で取引可能な例）

### 総当たりスキャン（推奨・n=1の落とし穴を避ける）

単一ペアの高相関は偶然かもしれない。`scan.py`は複数の経済市場×金融商品を
一括検証し、各ペアで**並べ替え検定**により「偶然この相関が出る確率（p値）」を
計算する。多重比較（たくさん試せば偶然の当たりが出る）も考慮し、
**偶然で期待される有意ペア数**と比較して結論を出す。

```bash
python scan.py                          # 既定の総当たり
python scan.py --n-perm 500 --max-lag 12 --save data/scan_results.csv
```

スキャンには必ず **BTC価格市場 × BTC現物** をポジティブコントロールとして
含める。ここで強い相関が出なければツール自体が壊れている合図なので、
他の「無相関」という結果も信用できない。

出力例（実測）:

```
経済市場ペア: 4件中 p<0.05 は 1件(偶然の期待値 ≈ 0.2件)。
うち『Polymarketが先行(lag>0)』の有意ペア = 0件  ← エッジ候補はこれだけ
→ 予測力の証拠なし。有意なペアがあっても方向が逆（金融商品がPolymarketに
  先行/同時）で、取引エッジにはならない。
対照実験(BTC): corr=+0.558 p=0.003 lag=+0 → パイプラインは正常に機能している。
```

**エッジになるのは「Polymarketが先行（lag > 0）」かつ有意なペアだけ**である点に注意。
有意でも金融商品側が先行しているなら、それは予測に使えない（後追い）。

## 結果の読み方

出力例:

```
データ点: 702 / 最大相関ラグ: -2(-2時間) / 相関係数: -0.177(弱い相関) / 同時刻の相関: +0.010
解釈: 金融商品がPolymarketより約2時間先行している(=Polymarketに予測の優位はない)。
```

- **最大相関ラグ > 0**: Polymarketが金融商品に**先行**している可能性（＝予測力の候補）
- **ラグ = 0**: 同時に動いている（先行関係なし）
- **ラグ < 0**: 金融商品の方が先行（Polymarketに優位なし）
- **相関係数の絶対値**: 0.1未満はほぼ無相関、0.3以上で意味を検討する価値

## 限界・注意（必ず読む）

- **n=1は偶然でありうる。** 単一市場・単一期間の結果は、複数市場・複数期間で
  再現しなければ意味がない。まずは色々な組み合わせで回して傾向を見ること
- **相関は因果ではない。** たとえ先行性が出ても、それが利益に変換できるかは別問題
  （取引コスト・スリッページ・約定タイミングを織り込むと消えることが多い）
- **経済的な結びつきのある組み合わせを選ぶこと。** 「Fed金利 × ドル円」のように
  イベントと商品に理屈のある関係が必要。無関係な市場同士の相関は無意味
- **Yahoo Finance / Polymarketの公開APIは仕様変更されうる。** 取得できなくなったら
  エンドポイントの変更を疑うこと
- このツールは**分析のみ**で、発注機能を一切持たない。合法な商品での実取引は
  楽天証券のRSS等、別途正規の手段で行うこと
- **現状、日本の金融商品に綺麗に対応づく流動性の高いPolymarket市場は少ない。**
  高出来高市場の多くは選挙・話題系で、ドル円・日経に理屈で結びつく市場は
  Fed金利など一部に限られる。スキャンで対象が数件しか出ないのはこのため
  （データ不足自体が1つの発見）
