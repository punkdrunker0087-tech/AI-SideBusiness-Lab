# マルチファクター投資フレームワーク

個別銘柄の将来を当てるのではなく、**長期に観測されてきたリスクプレミアム・
リターン特性を組み合わせる**という考え方の実装。5大ファクター（Value・
Momentum・Quality・Size・LowVol）を軸に、仮説設定から評価・露出分析・
相関・リバランス・パフォーマンス分解までを実データで実演する。

> 研究・学習用。発注機能は持たない。既存の `../signal-research/`
> （汎用シグナル研究）・`../aqm-strategy/`（具体戦略）と補完関係にある。

## 研究プロセスと実装の対応

| # | プロセス | 実装 |
|---|---|---|
| 1 | 仮説設定 | `hypothesis.py`（5ファクターの経済的根拠を先に固定） |
| 2 | データ準備 | `data_util.py`（価格・ファンダメンタル・セクター取得＋整合性確認） |
| 3 | ファクター定義 | `factors.py`（先読み防止を明示。価格系/スナップショット系を区別） |
| 4 | 評価 | `evaluation.py`（Information Coefficient・IC_IR・t値） |
| 5 | 組み合わせ | `correlation.py`（時系列相関・横断相関・冗長性） |
| — | 露出（エクスポージャー） | `exposure.py`（ファクター・セクター・集中度） |
| — | 相関と分散 | `correlation.py`（時間変化・ストレス時変化） |
| — | リバランス | `rebalance.py`（頻度別の回転率・コスト） |
| — | パフォーマンス分析 | `attribution.py`（市場・ファクター・銘柄固有・コストへの分解） |

## セットアップ & 実行

```bash
cd multifactor-investing
pip install -r requirements.txt
python demo.py
```

## ⚠️ 最重要の区別: 価格ベース vs スナップショットのファクター

| ファクター | データ源 | 過去バックテスト |
|---|---|---|
| **Momentum・LowVol** | 価格・出来高（全期間） | ✅ 先読みなく正式に評価可能 |
| **Value・Quality・Size** | Yahoo財務スナップショット（現在値のみ） | ❌ 過去への一律適用は先読みバイアス |

Value/Quality/Sizeは**ライブのエクスポージャー分析にのみ**正当に使う
（`../aqm-strategy/quality.py` と同じ制約）。本フレームワークは両者を
明確に分離し、スナップショット系を時系列ICで評価する誤りを構造的に防ぐ。

## 実測結果（25銘柄・5年）

### 評価（IC）
- **Momentum**: IC平均+0.015, t値+1.5（弱いが方向は仮説と一致）
- **LowVol**: IC平均-0.012, t値-1.1（弱く、方向は仮説と不一致）
- Value/Quality/Sizeは前述の理由で時系列IC評価を実施せず

### 相関と分散（時間変化）
Momentum × LowVolの相関は **全期間-0.37 → 直近-0.79**（以前-0.24から悪化）。
ストレス時（市場下位10%日）は**-0.30 → -0.45**とさらに強まる。
「平常時は分散されていても、ストレス時に関係が変わる」を実データで確認。

現時点の5ファクター横断相関ではMomentum×LowVolが**-0.77**と強い負の相関
（低ボラ銘柄は上がったばかりの銘柄とは違う、という構成上自然な関係）。
組み合わせる際はこの重複・裏返しの関係を意識する必要がある。

### リバランス（頻度別トレードオフ）

| 頻度 | リバランス回数(5年) | 年間回転率 | コスト負け |
|---|---|---|---|
| 週次 | 262 | 23.3 | -13.0%pt |
| 月次 | 61 | 11.8 | -6.5%pt |
| 四半期 | 21 | 6.3 | -3.7%pt |

頻度を上げるほど回転率・コストが増える教科書通りの関係が出た。
このユニバース・期間ではMomentumロング・ショート自体が正味マイナスで
あり、**「リバランスをどう最適化するか」より前に、Momentum単体では
市場ニュートラルαが出ていない**という点が優先度の高い問題（`../aqm-strategy/`
の結論と整合）。

### パフォーマンス分解
Momentumロング・ショートのリターンをmarket・LowVolへ回帰: **market β≈-0.0004**
（ロング・ショート構成によりほぼ市場中立、想定通り）。R²=0.14で大半は
α（切片、ここではマイナス）と残差として説明され、単純な線形分解では
捉えきれない要因が大きいことも示している。

## 実装中に見つけたバグ（教訓として記録）

`correlation.py`/`rebalance.py`（当初は `../aqm-strategy/strategy.py` にも
同型のバグがあった）で、月末・週末などの**暦日ラベル**をそのまま
`in close.index` で判定していたため、その日が土日祝で取引日でなければ
**リバランスが黙って抜け落ちる**バグがあった。週次では実質全滅していた
（`n_rebalances=0`）。日付そのものをresampleして実際の取引日を取得する形
（`rebalance_dates()`）に修正し、週次262回・月次61回・四半期21回という
5年間の理論値と一致する正しい回数になることを確認した。**「動いているように
見える」コードでも、回数の桁が理論値と合うかを検算することが発見の鍵だった。**

## 限界・注意

- 25銘柄の代理ユニバース。正式なファクター研究にはもっと広いクロスセクションが必要
- Value/Quality/Sizeは現在スナップショットのみ（点in-time履歴がない）
- 生存者バイアス未対応（無料データの制約）
- パフォーマンス分解は簡易線形近似（複利・非線形性は厳密に扱っていない）
- 発注機能なし
