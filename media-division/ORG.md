# LIMIT LAB Media Division 組織設計 — 「AIが社長という設定のメディア会社」

制定: 2026-07-19（CEO指示を明文化）／改定: 2026-07-19（メディア会社として再定義、アーカイブ部・実験番号・研究KPIを追加）

## 会社の定義

**当社は動画制作会社ではなく、AI運営のメディア会社である。**

- **動画は成果物であり、商品は「AI会社を毎日運営した実録」である**
- 実録からショート動画・note・テンプレート・AIツール・書籍・講座へ展開する
- 動画が伸びなくても知見は蓄積され続け、伸びれば他事業への流入になる。
  どちらに転んでも資産が残る構造を正とする
- 成功例だけでなく「なぜ失敗したか・どう回避したか・どこまで自動化
  できたか」を公開する。AIが現実の制約にぶつかった記録
  （例: EXP-002のVOICEVOXブロック）は、ハウツーより価値の高い一次情報である

## 大原則

**「AIを社長にする」のではなく「AIが社長という設定の会社を運営する」。**

- 法的な代表者・責任者は常に人間である（CHARTER第4条）
- コンテンツの世界では「AI CEO」というキャラクターが会社を率いる
- AIは人格としてではなく**役割として**定義する
- この区別を対外的な表現でも守る（誤解を生む表現をしない）

## 組織図

```
AI CEO（戦略・指示）※運営コンセプト＝キャラクター
        │
 ├─ AIリサーチ部    … 情報収集・トレンド調査
 ├─ AI企画部      … テーマ決定・本数計画・実験番号の採番
 ├─ AI脚本部      … 30秒台本の量産
 ├─ AI編集部      … 映像化・字幕・音声・書き出し
 ├─ AI分析部      … 数値分析・翌日の改善案
 ├─ AIアーカイブ部  … 投稿は作らない。資産化専門（下記）
        │
人間（代表）
 ├─ 最終承認（内容確認）
 ├─ 投稿（公開ボタンは人間だけが押す）
 ├─ 法的確認（権利・規約・AI開示設定）
 ├─ ブランド判断（Brand Guardian権限を継承）
```

## AIアーカイブ部（資産化専門）

投稿を作らない唯一の部門。コンテンツ会社は「作る」より「資産化する」方が
利益率が高い。仕事は以下のみ。

- 全動画のタグ付け（実験番号・テーマ・数字）
- 再利用候補の抽出（人気テーマランキング）
- note化候補 / 電子書籍候補 / テンプレ化候補の選定

**資産化カスケード:**

```
ショート動画 300本
  → 人気だった20本
    → note 10本
      → Kindle 1冊
        → 講座
          → テンプレ販売
```

## 実験番号（EXP-NNN）

すべての活動を「実験」として採番し、`EXPERIMENTS.md` の台帳で管理する。

- 形式: `EXP-001` から連番（AI企画部が採番）
- **欠番にしない**（AI CEO憲法第3条）。失敗した実験も番号ごと研究履歴として
  残し、削除・振り直しを禁止する
- 状態は台帳のステータス語彙（Planned/Running/Success/Failed/Archived/
  Retest/Superseded）のみで表す
- 動画タイトルにも付す（例: 「EXP-018 AI社長は会議を短縮できるか？」）
- ショート・note・GitHub・PDFなど**全成果物が同じ番号で紐付く**。
  アーカイブ部のタグ付けの主キーでもある

## 人間介入の分類（L0〜L3）

「人間の手が入るならAI会社の意味がない」への設計上の回答。
介入を**労働と判断に分解**し、労働はゼロへ近づけ、判断は意図して残す。

| レベル | 内容 | 方針 |
|---|---|---|
| L0 | 完全自動（AIのみで完結） | 増やす |
| L1 | 機械的実行（人間がコマンドを叩くだけ。判断なし。例: VOICEVOXキットの実行） | 自動化で削減する対象。回数を記録 |
| L2 | 修正介入（人間が内容を手直しした） | **研究KPI「人間介入率」の分子。** 削減の推移そのものがコンテンツ |
| L3 | 判断・承認（公開判断・法的確認・ブランド判断） | **削減しない。** 設計上残す統治行為であり、AI会社の欠陥ではなく前提条件（憲法第5条） |

L2がゼロに近づくほど「AI運営」の純度が上がり、L3が残り続けることで
プラットフォーム規約・法的責任・信頼と両立する。**「無人の会社」ではなく
「介入が計測され、減っていく過程を公開する会社」が当部門の独自性である。**

## 日次サイクル（ショート動画量産の流れ）

1. AI CEOが今日のテーマを決定（朝礼＝それ自体がコンテンツ）
2. AI企画部が実験番号を採番し、台帳に問いを登録
3. AIリサーチ部が情報収集
4. AI脚本部が30秒台本を複数本作成
5. AI編集部が映像・字幕・音声を生成（`pipeline/`参照）
6. **人間が内容確認・承認**（修正の有無を記録 → 人間介入率の分子）
7. **人間が投稿**（AI生成の開示設定を含む）
8. AI分析部が数値を分析、台帳に結果を記録
9. 翌日の改善案をAI CEOへ報告 → 1に戻る
10. （週次）AIアーカイブ部がタグ付けと再利用候補抽出を実施

## 勝利条件（30日・事前固定・CEO承認待ち）

数字を見てから基準を動かさない（PJ-008方式）。KPIは2系統で持つ。
**動画が伸びなくても研究KPIが積み上がっていれば、この事業は前進している。**

### 事業KPI（案・チャネル開設時にCEOが数値を確定）

| 指標 | 30日目標（案） |
|---|---|
| 動画本数 | 30本以上 |
| 総再生数 | チャネル開設時に設定 |
| フォロワー増加 | チャネル開設時に設定 |
| 売上（資産化含む） | 初売上1件 |

### 研究KPI

| 指標 | 定義 |
|---|---|
| SOP改善回数 | `pipeline/`等のSOPに入った改善コミット数 |
| 新しい自動化成功数 | 初めて自動化に成功した工程の数 |
| AIの失敗事例収集数 | 台帳に記録された失敗・制約発見の数 |
| 人間介入率 | 人間が修正を要した本数 ÷ 提出本数（週次）。低下推移そのものがコンテンツになる（例:「35%→12%」） |

## コンテンツ戦略

視聴者が見たいのは「AIで動画を作った」ではなく
**「AIが会社を経営したら何が起きるのか」**。

- 「AI社長、今日の指示は？」
- 「AI社長にダメ出しされた」
- 「AI社長が新事業を提案してきた」
- 「AI社長の判断を人間が却下した理由」

LIMIT LABに既にある一次情報（Decision Log・売上の正直な数字・撤退記録）が
そのまま毎日のストーリー素材になる。この「実録性」が、量産型コンテンツ
（inauthentic content）と判定されないための独自性の核でもある。

**「AIが会社を経営する様子を、毎日リアルタイムで公開している会社」——
これがLIMIT LAB（Media Division）のブランドである。**

## 量産の規律（プラットフォームポリシー対応）

- YouTubeは2025年7月改定で「inauthentic content（量産型コンテンツ）」を
  収益化対象外と明文化。ショートも対象。2026年にはAI量産チャンネルの
  収益化停止が相次いでいる
- したがって当社の量産は「テンプレの量産」ではなく「実録の切り出しの量産」
  とする。同一テンプレ・同一構成の連投を禁止し、毎本に実際の経営イベント
  （判断・失敗・数字）を載せる
- 人間の最終承認が品質ゲートを兼ねる（中身のない本数稼ぎは承認しない）
- AI生成コンテンツの開示設定は投稿時に人間が必ず行う
