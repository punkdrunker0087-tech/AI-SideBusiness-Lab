# LIMIT LAB Research Protocol — Media Division

**Version: v2026.1**（版管理ルールは下記「永遠のβ」参照）

**ブランドはLIMIT LABひとつ**（2026-07-19 CEO決定）。別会社名は立てず、
LIMIT LABの一部門「Media Division」として運営する。認知はすべて
LIMIT LABに積み上がる。

これは「OS」ではなく**研究プロトコル**である。何をするかの手順書ではなく、
何を検証し・何を記録し・どう終わるかを定めた実験計画として運用する。

## 永遠のβ（Continuous Experiment）

**この会社に完成版は作らない。**

- バージョンは `v<年>.<改定番号>` で刻む（v2026.1 → v2026.2 → v2027.1）。
  年内の大きな改定で数字を上げ、年が替わったら .1 に戻す
- 旧バージョンの記述は書き換えて消さず、gitの履歴とAnnual Reflectionに残す。
  「あの頃はこう考えていた」まで資産にする
- v1.0（完成）という概念を持たない。**Evidence Firstの組織は、完成した瞬間に
  改善が止まる。だからLIMIT LABは完成しない**
- バージョン更新はL4（創業者判断）である

当部門は**AI運営のメディア部門**である。動画は成果物であり、
**商品は「AI会社を毎日運営した実録」**。実録をショート動画・note・
テンプレート・書籍・講座へ資産化する。法人登記はしない（実法人化は
`CHARTER.md`の発動条件を満たした時のみ検討する）。

## LIMIT LABブランド内での位置づけ

```
LIMIT LAB（単一ブランド・人間のCEO）
 ├── Research Division … AI実験・検証・評価（既存LIMIT-OSの発掘フェーズ）
 ├── Media Division   … 実録の発信と資産化（この部門）
 └── Products Division … テンプレ・ツール販売（既存BOOTH/note事業）
```

「LIMIT LAB」を見れば、AI実験・AIツール・note・動画・テンプレートが
すべてつながる世界観にする。

- **憲法は共有**: LIMIT LABの`CONSTITUTION.md`（信頼と信用・先義後利・
  利用者第一・感動品質・技術は手段）をそのまま最上位規範として継承する。
  独自の憲法は作らない
- **知見も共有**: `knowledge/`（Decision Log・SOP・Institutional Memory）は
  グループ共通資産として参照・追記する
- **数字は分ける**: 売上・KPIはLIMIT LABと分けて記録し、会社ごとに
  Go/No-Go判断できるようにする
