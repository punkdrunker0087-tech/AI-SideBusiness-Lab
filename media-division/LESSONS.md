# LESSONS.md — 教訓ナレッジベース（EXP-010）

制定: 2026-07-24／改定: 2026-07-24（状態管理を追加。CEO指示）

## 位置づけ

- **実験結果そのもの** → `EXPERIMENTS.md`（EXP-NNN）
- **プロトコルレベルの単発観測** → `OBSERVATIONS.md`（OBS-NNN）
- **毎週の運用手順** → `WEEKLY.md` / `ROUTINE.md`
- **何度も使える教訓だけを、ここに昇格する**

研究は「実験 → レビュー → 教訓 → 次の実験」の循環になる。単なるログ
ではなく、研究所のナレッジベース。ここまで来ると、当部門は「動画生成
システム」ではなく「自ら学習し、運用方法まで改善していく研究システム」
に近づく。

## 状態（ステータス）

教訓は以下の4状態のいずれかを持つ。**「ルールを破った」ではなく、
ルールの例外が発生したこと自体を管理対象にする**——例外は隠すより
記録した方が研究として価値がある。

| 状態 | 意味 |
|---|---|
| Candidate | 提案されたが、Research Review未承認 |
| Accepted | Research Reviewで採択された正式な教訓 |
| Rejected | 採択されなかったが、履歴として残す |
| Superseded | より良い教訓に置き換えられた |

## 昇格ルール

1. **具体的な出来事に裏付けられていること**（`EXPERIMENTS.md`のEXP番号
   または`OBSERVATIONS.md`のOBS番号を出典として明記する。憶測を教訓に
   しない）
2. **特定の実験に限らない一般性を持つこと**（今後の別の実験・意思決定
   でも再利用できる知見であること）
3. **提案はいつでもできる**（Research Reviewの場を待たず、気づいた
   時点で`Candidate`として追記してよい）。**ただし正式な採択
   （`Accepted`への変更）は週次のResearch Review（「今週新しく作った
   ルールで、実際に運用してみると不都合が見つかったものはあるか？」
   および「研究プロセス自体の改善点」欄）でCEOが判断してから行う**
4. **教訓が後に反証された場合も削除しない。** `Superseded`として残し、
   反証した出典を追記する（AI CEO憲法第2条・第3条）

## 教訓一覧

### LESSON-001: 効果測定は、実装より先に測定手段の有無を確認する

状態: **Accepted**（2026-07-24・CEO提案時点で承認扱い）

背景アニメーション（EXP-007）の効果はYouTube Analyticsへの直接アクセス
がないため判定できない。実装を先に進めてから測定手段の欠如に気づいた。
次回以降、「効果を測る」実験は着手前に測定可能性を確認する。

出典: EXP-007

### LESSON-002: 運用手順は仕様書参照型にすると保守性が高い

状態: **Accepted**（2026-07-24・CEO提案時点で承認扱い）

トリガー本文に実行ロジックを直接埋め込むと、変更のたびに削除・再作成が
必要になり、「いつ・なぜ変えたか」の履歴が追跡しにくい。`ROUTINE.md`の
ような正典を1つ用意し、トリガー本文はそれを参照するだけにすると、
運用変更はファイル更新だけで完結する。

出典: EXP-009

### LESSON-003: 失敗はArchived/Supersededで残す、削除しない

状態: **Accepted**（2026-07-24・CEO提案時点で承認扱い）

中止した課題・棄却された仮説・反証された教訓は削除せず、ステータスを
変更して記録に残す。研究としての再現性・監査性を保つため。

出典: AI CEO憲法第2条・第3条、`REJECTION-LOG.md`の運用

### LESSON-004: Single Source of Truthは、導入しただけでは原則を満たさない

状態: **Candidate**（提案: 2026-07-24・AI自己観測。Day 7のResearch
Reviewで Accepted / Rejected / Superseded のいずれかに確定する）

Single Source of Truthを導入しても、その実装が本当に原則を満たしているか
（重複が生まれていないか）は別途レビューが必要である。`ROUTINE.md`
（正典）を作った当日に、Research Reviewの書式を`ROUTINE.md`と
`WEEKLY.md`の両方に書いてしまい、原則に反した。正典を作ること自体が、
重複を作らないことを保証しない。

出典: EXP-009（2026-07-24同日発見・自己観測）
