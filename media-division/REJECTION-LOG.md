# 反証ログ（REJECTION LOG）

AI CEOの提案のうち**採用されなかったもの**を記録する台帳。
採用した判断は台帳とDecision Logに残るが、棄却された仮説はここにしか残らない。
数ヶ月後に「今ならこの提案は成立する」と再評価するための資産である。

書式:

```
## Day NN（YYYY-MM-DD）
提案（AI CEO）: 朝礼を毎日2本に増やす
採用: No（判断者: 人間 / レベル: L3）
理由: L2介入率が上昇傾向だったため、生産量より品質改善を優先した
再評価条件: L2介入率が15%を下回って2週間安定したら
```

記録ルール:

- 提案の却下は失敗ではない。**却下の理由こそが観測データ**である
- 「再評価条件」を必ず書く（いつになったら読み返すべきかを未来に残す）
- 人間がAI CEOの提案を却下した場合も、AI CEOが自案を取り下げた場合も記録する
- 言葉は中立にする（成功を祝わない原則と同じ）
