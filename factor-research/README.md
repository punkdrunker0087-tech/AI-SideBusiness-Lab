# 学術的ファクター研究フレームワーク

「過去に高いリターンだった」ことより、経済的な説明・異なる市場/期間での
再現性・コストや制約を考慮した頑健性を重視する研究プロセスの実装。

> `../multifactor-investing/`（Value/Momentum/Quality/Size/LowVolの5
> ファクター・エクスポージャー分析）を補完する形で、本フレームワークは
> **明示的なユニバース構築基準・キャパシティ/市場インパクト分析・
> 包括的リスク評価・最終研究レポート生成**に焦点を当てる。

## 研究プロセスと実装の対応

| # | プロセス | 実装 |
|---|---|---|
| 2 | 投資ユニバース | `universe.py`（流動性・上場期間・データ品質を基準に判定し**構築ログ**を残す） |
| 3 | ファクター構築 | `factors.py`（Value/Momentum/Quality/Size） |
| 4 | リターン計算 | `returns.py`（日次/月次・adjcloseでコーポレートアクション調整済み） |
| 5 | リスク評価 | `risk_metrics.py`（Sharpe/Sortino/Calmar/下方偏差/VaR/CVaR） |
| 6 | 市場環境ごとの分析 | `regime_analysis.py`（強気/弱気レジーム別） |
| 7 | 実装可能性 | `capacity.py`（**平方根法則によるキャパシティ/市場インパクト分析**） |
| 8 | ポートフォリオ評価 | `portfolio_evaluation.py`（相関・分散化比率・回転率） |
| 9 | レポーティング | `report.py`（全セクションを1つのMarkdownレポートに集約） |

## セットアップ & 実行

```bash
cd factor-research

```

## 実測結果（30銘柄・3年）

### 2. ユニバース構築ログ
30候補すべてが基準（平均売買代金3億円以上・500営業日以上・欠損率2%
以下・株価100円以上）を満たし採用率100%。**採用/除外の判定過程を
毎回ログとして残す**ため、後から「なぜこの銘柄が入っているか」を
説明できる。

### 3-6. モメンタムのリターン・リスク・レジーム
- 年率リターン+8.4%、Sharpe+0.44、最大DD-23.8%
- **強気相場でSharpe+1.02、弱気相場で-1.27**——`signal-research/`
  で確認したモメンタム・クラッシュのパターンと整合する結果が
  別のユニバース・別の実装でも再現された

### 7. キャパシティ分析（本フレームワークの新規要素）

理論上のリターンは無限の資金を前提とするが、実際にはポジションを
建てるほど市場インパクト（`../execution-algorithms/`と同じ平方根法則）
が効いてくる。年率エッジ837bpsに対し:

| 侵食割合 | 到達するAUM |
|---|---|
| 10% | 約38.6兆円 |
| 25% | 検証範囲内(100兆円)で到達せず |
| 50% | 検証範囲内で到達せず |

超大型・超高流動性銘柄（平均売買代金63〜1,641億円）のみのユニバース
のため、キャパシティは非常に大きい。これは楽観的な誤りではなく、
**「どのユニバースを対象にするかでキャパシティの意味が全く変わる」**
ことを示す健全な結果——小型株中心のユニバースであれば、遥かに低い
AUMでエッジが侵食されるはずである。

### 8. ポートフォリオ評価
年率回転率16.3（月次フルリバランスのロング・ショートとして妥当な水準）。

### 9. レポート
`demo.py`実行で`factor_research_report.md`が生成され、上記すべてを
1つのMarkdownにまとめる。研究チェックリスト（仮説の事前定義・先読み
排除・複数環境検証・コスト考慮・統計的有意性検定の要否）を末尾に付す。

## 研究で重視される原則（このリポジトリでの対応）

| 原則 | 対応 |
|---|---|
| 仮説を先に定義する | 各ファクターの経済的根拠を明示（`factors.py`のdocstring） |
| 将来情報を排除する | モメンタムは`rolling`/`shift`のみ。Value/Quality/Sizeはライブ断面専用と明示 |
| 長期間・複数市場で検証する | レジーム別分析（`regime_analysis.py`）。複数市場は未対応（限界参照） |
| 取引コスト・流動性を考慮する | `capacity.py`のキャパシティ分析 |
| 統計的・経済的妥当性を評価する | `../backtest-framework/`のDSR/PBO/Reality Checkを別途適用推奨 |

## 限界・注意

- Value/Quality/Sizeは現在スナップショットのみ（点in-time履歴がない）。
  時系列でのリターン検証はMomentumのみ実施
- 単一市場（日本株）・30銘柄という小規模なユニバースでの検証
- キャパシティ分析は平方根法則という単純化モデル。実際のインパクトは
  銘柄・執行方法・市場構造で大きく変わる
- 統計的有意性の厳密な検定（DSR/PBO等）は別リポジトリ
  （`../backtest-framework/`）の適用が必要
- 発注機能なし
