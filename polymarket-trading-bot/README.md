# Polymarket 自動売買Bot（実験版）

`docs/articles/polymarket-prediction-markets.md` で調査したPolymarket APIを
使い、実資金で自動売買を行うためのBotの叩き台。

## ⚠️ 先に読むこと（免責・リスク）

- **実資金が減る可能性がある。** 予測市場は本質的に投機であり、戦略が
  過去データで機能しても将来の利益を保証しない。**投じた資金を失っても
  問題ない範囲でのみ使うこと。**
- **法規制・利用規約を自己責任で確認すること。** Polymarketは対象外の国・
  地域からの利用や、特定の属性のユーザーの利用を禁止/制限している
  （Terms of Serviceを参照）。日本居住者としての利用可否・税務上の扱い・
  金融商品取引法や賭博関連法規との関係は、このBotでは判断しない。
  **必要に応じて弁護士・税理士に確認すること。** 本リポジトリのコードは
  技術検証目的であり、法的助言ではない。
- **秘密鍵・APIキーは絶対にコミットしない。** `.env`はGit管理対象外
  （`.gitignore`参照）。秘密鍵はウォレットの資金移動権限そのものであり、
  漏洩は資金の完全な喪失に直結する。
- **デフォルトはドライラン（発注しない）。** 実際に注文を出すには
  `LIVE_TRADING=true` を明示的に設定する必要がある（下記参照）。

## 構成

| ファイル | 内容 |
|---|---|
| `bot.py` | メインループ。市場取得→戦略判定→（ドライラン or 発注） |
| `strategy.py` | 売買判断ロジック（モメンタム/しきい値の単純な例） |
| `risk.py` | 1回あたり上限額・1日あたり損失上限などのガード |
| `config.py` | 環境変数からの設定読み込み |
| `.env.example` | 必要な環境変数のテンプレート（実際の値は書かない） |
| `requirements.txt` | 依存パッケージ |

## セットアップ

1. Polymarketアカウントを作成し、Polygon上のUSDCで資金を供給する
2. `docs/articles/polymarket-prediction-markets.md` の手順でAPIキー
   （key / secret / passphrase）と、注文執行用のウォレット秘密鍵を取得する
3. 依存パッケージをインストール:

   ```bash
   cd polymarket-trading-bot
   pip install -r requirements.txt
   ```

4. `.env.example` を `.env` にコピーし、各値を埋める（**`.env`はコミットしない**）:

   ```bash
   cp .env.example .env
   ```

## 実行方法

### ドライラン（デフォルト・推奨: まずはこれで様子を見る）

```bash
python bot.py
```

発注は行わず、「どの市場に・いくらで・どちらに賭ける判断をしたか」を
ログに出力するだけ。

### ライブ（実資金で発注する）

`.env` で `LIVE_TRADING=true` を設定してから実行する。実行前に
`risk.py` の上限値（`MAX_ORDER_USD` / `MAX_DAILY_LOSS_USD`）を必ず
自分のリスク許容度に合わせて調整すること。

```bash
LIVE_TRADING=true python bot.py
```

## 戦略について

`strategy.py` の`simple_momentum_strategy`はプレースホルダーの単純な
モメンタム戦略（直近の価格変化率がしきい値を超えたら追随する）であり、
収益性は検証されていない。実運用前に、必ず`--dry-run`相当のログ出力を
一定期間観察し、過去データでのバックテストを行うこと（本リポジトリには
バックテスト機構は未実装）。
