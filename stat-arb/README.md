# 統計的アービトラージ研究フレームワーク

価格差そのものでなく、**複数資産の統計的な関係性**（コインテグレーション）
に着目する研究分野の実装。ペア候補の探索から検定・スプレッド分析・平均回帰
速度・レジーム監視・複数ペア管理までを実データで実演する。

> 研究・学習用。発注機能は持たない。

## 研究課題と実装の対応

| 研究課題 | 実装 |
|---|---|
| ペア候補の探索 | `pair_search.py`（同一業種・流動性・相関でスクリーニング） |
| コインテグレーション（二変量・多変量） | `cointegration.py`（Engle-Granger・Johansen） |
| スプレッドの分析 | `spread.py`（長期平均・分散・自己相関） |
| 平均回帰 | `spread.py`（半減期＝OU過程の離散近似） |
| レジーム変化 | `regime.py`（ローリング・コインテグレーション監視） |
| 複数ペアの管理 | `portfolio.py`（業種集中・共通リスク要因） |
| 売買戦略の検証 | `meanreversion.py`（Zスコア・ウォークフォワード） |

## セットアップ & 実行

```bash
cd stat-arb
pip install -r requirements.txt
python demo.py
```

## 実測結果（16銘柄・同業種ペア・5年）

### 1. 「相関が高い ≠ 長期的関係がある」を実証

| ペア | 相関 | コインテグレーションp値 | 判定 |
|---|---|---|---|
| 8306.T-8316.T（三菱UFJ-三井住友） | **0.895** | 0.100 | **× 非定常** |
| 8031.T-8058.T（三井物産-三菱商事） | 0.799 | **0.015** | **○ コインテグレーション** |
| 8001.T-6501.T（伊藤忠-日立） | 0.516 | **0.010** | **○ コインテグレーション** |

**相関0.895の銀行ペアがコインテグレーション棄却され、相関0.516の
ペアが有意**——相関の高さと長期均衡関係の有無は別物であることが、
まさに実データで示された。慎重な検定が必要という研究上の注意点そのもの。

### 2. 多変量（Johansen）

同業種4銘柄（商社3＋日立、業種分類の粒度上まとまった）でJohansen検定を
実施: **rank=1**（4銘柄中、1本のコインテグレーション関係を検出）。
二変量検定と整合的な結果。

### 3. スプレッド分析・平均回帰速度

8031.T-8058.Tのスプレッド: 標準偏差210.35、1次自己相関0.980（かなり
持続的）、**平均回帰速度θ=0.019・半減期36.7日**。「戻る」にしても
1ヶ月以上かかる遅い平均回帰であることが分かる。

### 4. レジーム監視で見えた「検定力」の教訓

同じペアの構造変化監視では「直近p値0.453・以前も0.473」で**両方とも
非定常**と出た。全期間（約1220日）の検定ではp=0.015で有意だったのに、
ローリング窓（120日）では有意にならない ―― これは矛盾ではなく、
**ADF検定はサンプルサイズが小さいと検出力が落ちる**という統計的に
正しい現象。「関係があるはず」でも短い窓では検出できないことがある、
というレジーム分析の限界を示す実例になった。

### 5. ウォークフォワード売買（四半期ごとにヘッジ比率再推定）

| ペア | 取引回数 | 総PnL(粗) | コスト | 総PnL(純) |
|---|---|---|---|---|
| 8031.T-8058.T | 85 | +1721 | -655 | **+1066** |
| 8001.T-6501.T | 83 | +1892 | -269 | **+1623** |

両ペアともコスト差引後もプラス。ただし取引頻度が高く（5年で83-85回）
コストの影響も大きい（前者はコストが粗利益の38%を吸収）。

### 6. 複数ペア管理

採用した2ペアはどちらも**Conglomerates（商社・総合電機）業種に集中**
していると自動検出。ペア間PnLの相関は**+0.48**で、個々は市場中立でも
ポートフォリオ全体では共通リスク要因に束ねられている可能性を明示。

## 先読み防止の設計

- **ヘッジ比率**: 全期間で1回推定するのではなく、`meanreversion.
  walk_forward_hedge_ratio()`が四半期ごとに**過去データのみ**で再推定
- **Zスコア**: ローリング窓（20日）の平均・標準偏差のみを使用（未来を見ない）
- **コインテグレーション検定そのもの**は全期間データを使うため、これ自体は
  「その期間に関係があったか」の事後検定。将来も続くかは`regime.py`で別途監視

## 限界・注意

- 16銘柄・同業種ペアという小規模ユニバースでの実演
- 業種分類が粗く（Yahoo `assetProfile`のindustry）、商社と日立が同じ
  "Conglomerates"に分類されるなどの粗さがある（実データの品質課題そのもの）
- コインテグレーションは事後検定。将来の関係継続を保証しない
- 半減期36.7日は長く、実務上のポジション保有コスト（金利・借株料）を
  考慮していない
- 発注機能なし
