# SOP — 標準作業手順書

Fable-5クラスのモデルが確立した作業手順を、**後続のどのAIモデルでも
同じ品質で再現できる**ように書き起こした手順書群。モデルの性能差を
「手順の具体性」で吸収することが目的である。

## 書き方のルール

- 判断基準は数値・チェックリストで書く（「良い感じに」と書かない）
- コードの型はスケルトンとして貼る（ゼロから設計させない）
- 過去の失敗（`knowledge/Institutional Memory/what-failed.md`）への
  参照を必ず含める（同じ穴に落ちないため）
- 新しいSOPを追加したら`LIMIT-OS.md`に一行追記する
