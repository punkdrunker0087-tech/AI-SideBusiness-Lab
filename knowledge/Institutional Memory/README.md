# Institutional Memory — 組織知

`knowledge/Decision Log/`が「なぜその経営判断をしたか」を記録するのに
対し、こちらは**現場レベルで再利用できる知見**を蓄積する層である。
「このプロンプトはよく効いた」「この販売ページはCVRが高かった」
「このワークフローは失敗した」といった、次の案件・商品にすぐ使える
実践知を記録する。

## ファイル構成

- `what-worked.md` — 再現すべき成功パターン
- `what-failed.md` — 繰り返さない失敗パターン

## フォーマット

| 日付 | カテゴリ | 内容 | 根拠となる数字・事実 | 再利用可否 |
|---|---|---|---|---|

- **カテゴリ**: Prompt／Workflow／販売ページ／クリエイティブ（サムネイル等）／
  価格設定／チャネル選定 など
- **根拠となる数字・事実**: 実際のCVR・売上・反応数など（`docs/daily-metrics.md`
  や`knowledge/Customer Voice/`と連動）。数字がない場合は「未計測」と
  明記し、印象だけで「効いた」と書かない
- **再利用可否**: そのまま使える／条件付きで使える／この案件限定

## 運用ルール

- 市場調査部・制作部・マーケティング部は、新しい案件に着手する前に
  `what-worked.md`を確認し、既存の知見を活かせないか検討する
- 記録は実際にあった事実のみとする。まだ実績データがない場合は空欄の
  ままにし、architecture上「まだ知見がない」ことが分かる状態にする
