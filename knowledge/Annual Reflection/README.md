# Annual Reflection — 年次レビュー

数字（売上・KPI）だけでなく、「Purposeにどれだけ近づけたか」を年に
一度振り返る。会社発足日（2026-07-04、BOOTHショップ「LIMIT LAB WORKS」
公開日）を基準に、毎年その前後で実施する。

## 記録先

`knowledge/Annual Reflection/<年>.md` を年次で1ファイル作成する。

## フォーマット

```
# <年> Annual Reflection

今年最も価値を届けた商品は何か:
- <商品名と、なぜそう言えるか（Customer Voice・売上等の根拠）>

最も大きな学びは何か:
- <knowledge/Lessons Learned/・knowledge/Decision Log/を踏まえて>

理念に最も沿った判断は何だったか:
- <Value Score・Brand Guardianが発動した場面など、CONSTITUTION.mdに
  照らして誇れる判断>

来年は何を目指すか:
- <Purpose/Mission/Visionに対する来年の重点>
```

## 運用ルール

- 初回のAnnual Reflectionは2027年7月前後（会社発足から1年）に実施する
- 数字だけで判断せず、`CONSTITUTION.md`のPurpose・Vision・Values に
  照らして書く
- 過去の`knowledge/CEO Weekly Review/`・`knowledge/Performance Review/`・
  `knowledge/Decision Log/`を集約する形で作成してよい
