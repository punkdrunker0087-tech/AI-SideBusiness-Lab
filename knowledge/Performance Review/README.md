# Performance Review — AI社員の月次振り返り

目的は評価（誰が優秀か）ではない。**同じ失敗を繰り返さないこと**である。
月次で、各部門（市場調査部・検証部・事業評価部・法務部・制作部・
マーケティング部・分析部・改善部）およびCEO・PMについて振り返る。

## 記録先

`knowledge/Performance Review/<YYYY-MM>.md` を月次で1ファイル作成する。

## フォーマット（部門ごとに）

```
### <部門名>

成果:
- <その月に達成したこと>

ミス:
- <失敗・見落とし・手戻りが発生したこと>

改善:
- <ミスに対して実際に行った改善（Marketing.mdやdecision-framework.md
  の更新など、具体的な変更点にリンクする）>

次月の目標:
- <次月に注力すること>
```

## 運用ルール

- 実施タイミング: 月末（`knowledge/CEO Weekly Review/`の月内最終回と
  合わせてよい）
- 各部門の「ミス」は、`cases/<ID>/`の実際の記録やLessons Learnedを
  根拠にする。架空の失敗を作らない
- 「改善」は、実際にこのリポジトリに反映された変更（コミット・ドキュメント
  更新）を指すこと。反映されていない改善案は「次月の目標」に書く
- 初回のPerformance Reviewは、会社発足（2026-07-04 BOOTHショップ公開）
  から丸1ヶ月経過後（2026-08-04以降）に実施する
