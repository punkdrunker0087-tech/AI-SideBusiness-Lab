# Decision Log — 経営判断の記録

CEOが下した重要な経営判断を、根拠とともに記録する。「なぜこの判断を
したのか」を半年後・1年後でも追えるようにするための資産であり、
AIエージェントが将来同種の判断をする際の参考データにもなる。

## 記録するタイミング

- 新規案件のGo/No-Go・投資強度の決定
- 価格・商品ラインナップの変更
- ブランド・戦略方針の転換
- 組織構造・ルールの新設／改廃
- Value ScoreやBrand Guardianの拒否権が行使された判断

## フォーマット

各判断を `knowledge/Decision Log/<日付>-<短い件名>.md` として1ファイル
ずつ記録する。

```
# <件名>

日付: YYYY-MM-DD
決定: <何を決めたか>

## 根拠
- <市場調査・データ等の根拠1>
- <根拠2>
- <根拠3>

## 承認
CEO承認: <はい/条件付き/保留>
```
