# 開発憲章と実験計画書 — 磁石協力ゲーム（仮）

> **このドキュメントに書かれた内容は、信念ではなく仮説である。**
> **仮説は、実験によっていつでも修正・棄却される。**

これはGDD（ゲームデザインドキュメント）ではない。
「何を信じていて、何をまだ疑っているか」を整理した、実験ログの起点である。

---

## 目的

このプロジェクトの目的はゲームを完成させることではない。
**「何が面白いのか」を実験によって特定すること**である。

「面白いゲーム」を作るのではなく、「面白さを発見する」。
アイデアを証明しようとするのではなく、アイデアを壊しにいく。
壊れずに残ったものだけを、ゲームの柱にする。

---

## 開発原則

### 原則1：1ビルド = 1仮説
一つのビルドでは一つの問いしか検証しない。

### 原則2：棄却条件は実装前に書く
作ってから評価基準を決めない。
事後的に「まあ、それなりに面白い気もする」と合格させることを防ぐ。

### 原則3：原因が分からないビルドを作らない
複数の新要素を同時に追加しない。
面白くなかったとき、原因候補が一つに絞れる状態を保つ。

### 原則4：ルールより先に、おもちゃを作る
パズルやステージより、触って楽しいかを先に確認する。
事故が先、ルール理解は後。

### 原則5：理論より観察を優先する
美しい説明より、プレイヤーが実際に何をしたかを信じる。
「反転が核である」のような綺麗な理論ほど、交絡を疑う。

## 現時点の仮説

| ID | 仮説 | 状態 |
|----|------|------|
| H1 | 磁石は触っているだけで楽しい | 未検証 |
| H2 | 事故は笑いを生む | 未検証 |
| H3 | 情報の非対称性は会話を生む | 未検証 |
| H4 | 反転は笑いではなく、事故を情報へ変える | 未検証 |
| H5 | 能力は個人ではなく、関係性から生まれる | 未検証 |

※ H5は一番最後まで仮説扱いとする。
最も美しく、最もタイトルに近い理論であるがゆえに、最も慎重に疑う。

### 仮説の背景（議論の要約）

- **H1**：磁石には「引き寄せられて吹っ飛ぶ」瞬間の面白さ＝おもちゃとしての価値が独立に存在するはず。ただし現実の磁力（距離の2乗〜4乗で減衰）は体感が「何も起きない→突然スナップ」になる。面白い力の曲線は物理法則ではなく設計変数。
- **H2**：笑いを駆動するのは「協力の最中」「物理的に派手」「犯人がいる」の3点という予想。天災（誰のせいでもない事故）より仲間起因の事故が強いはず。
- **H3**：面白さの源泉は磁石そのものではなく「自分は知らない・仲間と話すと分かる・試して初めて確信できる」という流れ、という当初仮説。ただし測るべきは会話量ではなく会話の質（＝他プレイヤーの行動を変えた発話）。
- **H4**：反転（引力⇄斥力の切り替わり）は笑いの発生装置ではなく「究明すべき謎を残す事故」を作る装置、という予想。ハンマー事故は「wwww」で終わり情報を残さない。反転事故は「変わったの俺？お前？」を残す。
- **H5**：磁気の世界は全員の極性を一斉反転しても観測結果が変わらない。「誰がNか」には答えが存在せず、意味があるのは「誰と誰が同極か・異極か」という関係だけ。個人の正体は原理的に存在しない。

## Toyロードマップ

### Toy 0

**問い**：磁石は触るだけで楽しいか（H1）

| 項目 | 内容 |
|------|------|
| 内容 | プレイヤー2〜4人・磁石・狭い橋・崖。それだけ |
| 可変 | 磁力曲線・強さ・効き始める距離（実行中にスライダーで変更可能にする） |
| 固定 | ローカル（1台のPCにゲームパッド2〜4本）・地形 |
| 測定 | 事故が自然に起きるか／何度も触りたくなるか／「もう一回」が出るか |
| 規模 | 3時間ビルド＋パラメータUI 1時間 |

**棄却条件（実装前に確定）**：
15分遊んで、誰からも「もう一回」が自発的に出なければ棄却。
パラメータ探索を経てもダメなら、磁石という題材ごと再考する。

### Toy 1

**問い**：事故は笑いになるか（H2・H4）

| 項目 | 内容 |
|------|------|
| 追加 | 変換ボタン・ハンマー比較・即時復帰・重い鉄球（一人では動かない） |
| 測定 | 笑い／犯人探し（責任転嫁）／事故後の行動 |

**A/B/C事故比較（H4の切り分け）**：
- テストA：反転あり・事故あり（変換装置で引力が斥力に）
- テストB：反転なし・事故あり（**仲間がボタンを押すとハンマーが飛ぶ**。天災ではなく仲間起因にして「犯人の有無」の交絡を排除する）
- テストC：反転あり・事故なし

測定は2軸で取る：**笑いの量**と**事故後30秒の発話内容**。
予言：笑いの量では反転あり/なしに大差は出ない。発話内容では決定的な差が出る。
外れたらH4を棄却する。

**棄却条件（実装前に確定）**：
事故が起きても誰も笑わず、復帰後に同じ場所へ戻らなければ棄却。

### Toy 2

**問い**：非対称性は会話を変えるか（H3）

| 項目 | 内容 |
|------|------|
| 追加 | 極性非表示・ラベルなし固定磁石柱（一人で相対情報は得られるが確定はできない） |
| 測定 | 発話内容／自発的実験／仮説形成 |

**A/Bテスト**：極性表示ON/OFFをデバッグトグルで切り替え、別グループで比較。
差が出なければH3を棄却。

**評価基準（30/60/90）**：
1. 30秒以内に最初の事故は起きたか
2. 60秒以内にプレイヤーが自分たちで遊び方を発明したか
3. 90秒以内に「もう一回！」が自然に出たか

**診断表（基準を落とした場合に回すダイヤル）**：
- 30秒で事故が起きない → 磁力の強さ・地形の狭さ
- 60秒で遊びが発明されない → 地形のアフォーダンスが読めていない
- 90秒で「もう一回」が出ない → 復帰の速度・再試行の摩擦

**棄却条件（実装前に確定）**：
極性を隠しても会話量・内容がToy 1と変わらなければ、非対称性の柱を降格する。

### テストプレイの運用ルール

- 開発者はテスト中、絶対に喋らない。ヒントを一言でも出した瞬間、「遊びの発明」は測定不能になる。黙って録画する。
- 発話は「仮説」「指示」「驚き」「責任転嫁」の粗いタグで数える。価値ある発話＝他のプレイヤーの行動を変えた発話。
- 笑いの直前に何が起きたかをログする（おもちゃ層の性能測定）。
- セッション終了後の雑談を観察する。ルールの話をしていたら謎ドリブン、事件の話をしていたらパーティードリブン。どちらの方向に育てるかの最重要データ。

## 決定事項（仮説ではなく、決めたこと）

- **エンジンはGodot**。基準は完成度ではなく試行回数。プロトタイプは70%の確率で作り直す前提であり、捨てやすさを優先する。Unityへの移行はToy 2成功後、オンライン同期・Steamを見据えた段階で判断する。
- **Toy 0〜2はローカル・カウチプレイのみ**。オンライン同期は作らない。同じ部屋で遊ばせれば、評価指標（笑い・会話）がその場で全部観測できる。ローカルは安いだけでなく、測定器として上位。
- **UIに "N" "S" は絶対に出さない**。開発者だけがデバッグ画面で見る。プレイヤーが「反発組」「白柱チーム」と勝手に名付ける余地を残す。ゲームが名前を与えないから、プレイヤーが名前を作る。
- **一人で完結できる検証を作らない**（ただし一人でもヒントは得られる）。ラベル付き磁石・自分の極性が映る鏡のような「便利ギミック」は、会話の必然性を殺す。

## 反証ノート

失敗したらどうするかを、先に書いておく。サンクコストに引きずられないために。

**もしToy 0で磁石自体が面白くなければ：**
- ゲームを終了する。
- 「情報の非対称性」は別テーマで再利用する。

**もしToy 1で事故が笑いにならなければ：**
- おもちゃ層（物理の手触り・復帰速度）を疑う。パーティーゲームとしての土台がないままToy 2へ進まない。
- 反転（H4）の検証は、事故が笑いになる土台の上でしか成立しない。

**もしToy 2で極性を隠しても会話が変わらなければ：**
- 非対称性を柱から降格する。
- 純粋な物理パーティーゲームへ方向転換する。

**もしH5（関係性）がプレイヤーの発話に一切現れなければ：**
- 事故後の会話が「俺が変わった」（個人の話）ばかりで「俺とお前が逆になった」（関係の話）が出ないなら、「関係性」は開発者にだけ見えている美しい理論だったと認める。
- タイトル・コンセプト文からも関係性の語を外す。

## 付録：記憶に残る30秒（未検証のビジョン）

> 崖の上の細い橋。AとBは引き合う性質を利用して、互いを支えにしながら渡っている。
> 中盤、Cが善意で（あるいは面白半分で）変換装置を押す。
> 次の瞬間、引力が斥力に反転し、Bが吹っ飛んで崖下へ。全員爆笑。
> Bが復帰しながら叫ぶ——「待て、今ので分かった。**変わったのはお前じゃなくて俺だ！**」

この30秒は「事故で笑う」「法則を発見する」「関係性を理解する」が一発で重なる瞬間として書かれた。
ただし原則5に従い、これは目指す絵であって証明された核ではない。
Toy 1のA/B/C比較が、この絵のどの成分が本当に効いているかを教えてくれる。
