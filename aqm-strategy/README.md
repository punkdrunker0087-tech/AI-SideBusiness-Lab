## ユニバース拡大（25→225銘柄）で分かったこと（AQM-01-01の検証結果）

以前の25銘柄検証で「敗因はQualityデータの欠如ではなく、ユニバースが
小さすぎる可能性が高い」という仮説を立てた。日経225（225銘柄）へ
拡大して**その仮説を実際に検証**した結果:

| | 25銘柄（旧） | 225銘柄（新） |
|---|---|---|
| 総リターン | -47.6% | **-2.9%** |
| Sharpe | -0.72 | **-0.00** |
| 対TOPIXベータ | -0.05 | -0.00 |
| `--quality-static`（先読み上限推定）総リターン | -39% | **-17.7%** |

**ユニバース拡大により、絶対リターン・Sharpeとも大幅に改善した**
（-47.6%→-2.9%）が、**それでも正のαには届かなかった**。さらに、
先読みバイアスありの最も有利な条件（`--quality-static`）ですら225銘柄
時代でも-17.7%とマイナスのままだった。

**結論（仮説の検証結果、断定は避ける）**: 「ユニバースが小さすぎたこと」
は絶対リターンの悪化に確かに寄与していた（縮小改善という形で裏付けられた）
が、**それだけが敗因ではない**。225銘柄・先読みバイアスありの条件でも
なお勝てないことから、**Momentum/LowVol/Liquidityというファクター定義
自体、またはこの検証期間（2021-2026、TOPIXが+136%上げた一貫した強気
相場）がロング/ショート market-neutral戦略に不利だった可能性**が、
ユニバースサイズと並ぶ、あるいはそれ以上の要因として浮上した。

## ファクター定義・パラメータのグリッドサーチ + DSR/PBO検証（`factor_grid_analysis.py`）

225銘柄でも絶対リターンが正にならなかったことを受け、「パラメータの
選び方次第で勝ち筋がないか」を調べた。momentum_lookback(60/120/252日)×
top_q(0.1/0.2/0.3)×rebalance(月次/週次) の**全18通り**をグリッドサーチし、
`../backtest-framework/stats_eval.py` の Deflated Sharpe Ratio・PBOで
「グリッド内最良の結果が本物か、単なる多重検定の偶然か」を判定した。

```
最良パラメータ: momentum_lookback=252  top_q=0.1  rebalance=ME
  観測Sharpe(年率換算)=0.333  試行数=18
  期待される最大Sharpe(偶然・年率換算)=0.645
  Deflated Sharpe Ratio(DSR) = 0.247
  PBO(過学習確率) = 0.032
判定: DSR・PBOの基準を満たさない → 多重検定による偶然の可能性が高い
```

**グリッド内最良（252日モメンタム・上下10%・月次リバランス、Sharpe
+0.33）ですら、18通り試せば偶然に出てくると期待される最大Sharpe
（+0.645）を下回っていた。** つまりこの好成績は「本当に良いパラメータを
見つけた」のではなく、**18回試したことで生じた選択バイアスの範囲内**
という判定になる。

PBOが低い（0.032）点は一見矛盾するように見えるが、PBOは「IS最良が
OOSで中央値以下に落ちる頻度」を測るもので、DSRとは問いが異なる
（「このパラメータがIS/OOS間で相対的に安定して上位か」であって、
「絶対的に良い」かではない）。**両指標は補完的であり、DSRが低い時点で
「グリッド内最良を信頼できるα」として採用すべきではない**、という
判断を優先した。

## 複数レジームでの検証（15年・2011-2026、`--range 15y`）

単一の強気相場（2021-2026）だけでは評価が偏るため、2011年（東日本大震災後）
から現在までの15年、Abenomicsの大相場・2015-16年の停滞・2018年の下落・
2020年のコロナショックと急回復・2022年の調整・直近の強気相場と、複数の
レジームを含む期間で再検証した。

⚠️ **この検証中に別のデータ品質バグを発見・修正した**: `1306.T`
（TOPIX連動ETF）は2015年1月に10:1の株式分割を行っているが、Yahoo
FinanceのadjcloseはこれをRETROACTIVEに調整しておらず、
2014-12-30(986円)→2015-01-05(98円)という**偽の90%急落**が15年データに
混入していた（5年データではこの分割日を含まないため露見していなかった）。
`panel.py`に`_fix_unadjusted_splits`を追加し、持続的なレベルシフト
（一時的なグリッチとは異なり新水準がその後も続く場合のみ）を検出して
遡及調整するようにした。実装中、検出ロジックの順序を誤ると、別の
一時的な2日グリッチ（2026年3月30-31日、翌日には元の水準に戻る）の
「回復の一歩」を誤って「本物の分割」と誤認し、15年分の履歴全体を
誤って10倍にスケーリングしてしまう副作用も発見し、「先に1日グリッチ
除去→後で持続的な分割検出」の順に直して修正した。

修正後の実測結果:

```
戦略      : 総リターン -55.6%  CAGR -5.4%  Sharpe -0.41  最大DD -66.6%  β -0.09
TOPIX(B&H): 総リターン +639.5% CAGR +14.6% Sharpe +0.79  最大DD -32.7%
対TOPIXベータ = -0.09 → ○ ほぼ市場ニュートラル
```

**単一の強気相場だけの問題ではなかったことが分かった**: β≈0の市場

機能する」という単純なストーリーも成り立たない。

一方で、**直近2024-2026年の3年間は強気相場にもかかわらず戦略が
プラス**（+2.6%・+6.4%・+2.9%）という、2012-2023年の12年間とは異なる
パターンが見えた。これは東証の資本効率改革（PBR1倍割れ是正要請、
2023年〜）以降の日本株市場構造の変化を反映している可能性はあるが、
**3年間・プラス幅も小さいというサンプルの小ささから、これを「戦略が
効き始めた」と断定するのは時期尚早**。今後の推移を見て判断すべき
観察事項として記録するに留める。

## 今回の到達点と正直な結論

- **設計を実現できた**: β≈0の市場ニュートラルL/Sを、コスト・空売り・分割調整・
  異常値除去まで含めて実装。さらに**Yahoo財務でQualityを取得し、4ファクター版の
  ライブランキング（設計の完全形）を実現**した
- **ユニバース拡大（25→225）で絶対リターンは大幅改善したが、依然マイナス**。
  「ユニバースが小さすぎた」という仮説は部分的に正しかったが、それだけが
  敗因ではなかった
- **含意**: J-Quantsで点in-time Qualityを苦労して統合する前に、ファクター定義
  自体（Momentum/LowVol/Liquidityの重み・定義）や、より長期・複数レジームを
  含む検証期間の見直しを優先すべき
- **パラメータのグリッドサーチでも勝ち筋は見つからなかった**: 18通りの
  パラメータの中で最良だったもの（Sharpe+0.33）ですら、DSR=0.247・
  「18回試せば偶然出てくると期待される最大Sharpe(+0.645)」を下回り、
  **多重検定による偶然と統計的に区別できなかった**。パラメータチューニング
  で解決する問題ではないことが分かった
- **単一の強気相場が原因という仮説も否定された**: 15年・複数レジーム
  （Abenomics相場・停滞・弱気・コロナショック・調整・直近強気）で
  再検証したが、15年中11年がマイナスで、β≈0の市場ニュートラル性は
  レジームを問わず維持される一方、絶対リターンはレジームに関わらず
  概ねマイナスだった。ただし直近3年（2024-2026）だけは強気相場でも
  プラスという観察はあり、断定はできないが記録した

## 限界・注意

- 代理225銘柄（日経225）は**正式なTOPIX500ではない**（大型株偏重・
  生存者バイアスもあり、TOPIX500が持つ中小型株のクロスセクションを欠く）
- 財務データ非対応のため Quality は欠落（設計の中核が未実装）
- 日足・終値ベースの簡易執行モデル。実運用のスリッページ・板の薄さは未反映
- 15年（2011-2026）の複数レジームで検証済みだが、日経225の225銘柄という
  同一ユニバース・同一ファクター定義のままなので、ファクター設計自体の
  限界までは切り分けられていない
- Yahoo Financeのadjcloseは分割・併合を必ず遡及調整しているとは限らない
  （実例: `1306.T`の2015年10:1分割）。`_fix_unadjusted_splits`で持続的な
  レベルシフトのみ検出・修正しているが、検出漏れ・誤検知のリスクは
  完全には排除できない
- **このツールは検証のみ**。実発注機能は持たない
