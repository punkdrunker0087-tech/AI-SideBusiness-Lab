## 結果の読み方

market-neutral戦略の本当のベンチマークは **TOPIXではなく「現金(0%)」**。
市場に勝つのではなく、**市場と無相関（β≈0）に正のリターン**を出せるかを見る。

```
戦略      : 総リターン  -2.9%  CAGR  -0.6%  Sharpe -0.00 ... β -0.00
TOPIX(B&H): 総リターン +136.3% CAGR +19.4% Sharpe +1.02 ...
対TOPIXベータ = -0.00 → ○ ほぼ市場ニュートラル
判定: × market-neutralな優位は確認できない
```

> 数値は2026-07-22、ユニバースを25→225銘柄（日経225）へ拡大した後のもの。
> リバランス日付選定バグ修正（暦日ラベルが取引日でない月は黙ってリバランスが
> 抜けていた。`strategy.py`参照）は既に反映済み。

- **β≈0 は設計通り**（ドルニュートラル構築が機能）。ユニバース拡大（25→225）で
  β自体は変わらず（-0.05→-0.00）ほぼ完全な市場ニュートラルを維持した
- **絶対リターンは依然マイナス**（-2.9%、25銘柄時代の-47.6%からは大幅に改善したが
  正のαではない）→ v0（Quality抜き）ではαは出ていない
- TOPIXが大きく上げていることは無関係（market-neutralは市場方向を取らない）

## Qualityデータ源の調査結果（実地検証）

「試せるものは全部試す」で財務データ源を検証した:

- **Yahoo quoteSummary（crumb認証）**: ✅ **登録不要で225銘柄のROE・粗利率が
  取れた**。資本集約度はDuPont分解（利益率÷ROA=資産回転率、その逆数）で近似。
  → `--live` の4ファクター・ライブランキングとして実装済み（先読みなしの正当な用途）
- **J-Quants API 無料プラン**: 過去2年・12週遅延・要メール登録。**点in-timeに近い
  履歴**が取れる唯一の現実的な無料手段だが、登録が必要
- **EDINET API**: フル履歴だがAPIキー＋XBRL解析が重い
- **SimFin / Alpha Vantage**: 日本株カバレッジ不明

## ユニバース拡大（25→225銘柄）で分かったこと（AQM-01-01の検証結果）

以前の25銘柄検証で「敗因はQualityデータの欠如ではなく、ユニバースが
小さすぎる可能性が高い」という仮説を立てた。日経225（225銘柄）へ
拡大して**その仮説を実際に検証**した結果:

| | 25銘柄（旧） | 225銘柄（新） |
|---|---|---|
| 総リターン | -47.6% | **-2.9%** |
| Sharpe | -0.72 | **-0.00** |
| 対TOPIXベータ | -0.05 | -0.00 |
| `--quality-static`（先読み上限推定）総リターン | -39% | **-17.7%** |

**ユニバース拡大により、絶対リターン・Sharpeとも大幅に改善した**
（-47.6%→-2.9%）が、**それでも正のαには届かなかった**。さらに、
先読みバイアスありの最も有利な条件（`--quality-static`）ですら225銘柄
時代でも-17.7%とマイナスのままだった。

**結論（仮説の検証結果、断定は避ける）**: 「ユニバースが小さすぎたこと」
は絶対リターンの悪化に確かに寄与していた（縮小改善という形で裏付けられた）
が、**それだけが敗因ではない**。225銘柄・先読みバイアスありの条件でも
なお勝てないことから、**Momentum/LowVol/Liquidityというファクター定義
自体、またはこの検証期間（2021-2026、TOPIXが+136%上げた一貫した強気
相場）がロング/ショート market-neutral戦略に不利だった可能性**が、
ユニバースサイズと並ぶ、あるいはそれ以上の要因として浮上した。

## 今回の到達点と正直な結論

- **設計を実現できた**: β≈0の市場ニュートラルL/Sを、コスト・空売り・分割調整・
  異常値除去まで含めて実装。さらに**Yahoo財務でQualityを取得し、4ファクター版の
  ライブランキング（設計の完全形）を実現**した
- **ユニバース拡大（25→225）で絶対リターンは大幅改善したが、依然マイナス**。
  「ユニバースが小さすぎた」という仮説は部分的に正しかったが、それだけが
  敗因ではなかった
- **含意**: J-Quantsで点in-time Qualityを苦労して統合する前に、ファクター定義
  自体（Momentum/LowVol/Liquidityの重み・定義）や、より長期・複数レジームを
  含む検証期間の見直しを優先すべき
