## ユニバース拡大（25→225銘柄）で分かったこと（AQM-01-01の検証結果）

以前の25銘柄検証で「敗因はQualityデータの欠如ではなく、ユニバースが
小さすぎる可能性が高い」という仮説を立てた。日経225（225銘柄）へ
拡大して**その仮説を実際に検証**した結果:

| | 25銘柄（旧） | 225銘柄（新） |
|---|---|---|
| 総リターン | -47.6% | **-2.9%** |
| Sharpe | -0.72 | **-0.00** |
| 対TOPIXベータ | -0.05 | -0.00 |
| `--quality-static`（先読み上限推定）総リターン | -39% | **-17.7%** |

**ユニバース拡大により、絶対リターン・Sharpeとも大幅に改善した**
（-47.6%→-2.9%）が、**それでも正のαには届かなかった**。さらに、
先読みバイアスありの最も有利な条件（`--quality-static`）ですら225銘柄
時代でも-17.7%とマイナスのままだった。

**結論（仮説の検証結果、断定は避ける）**: 「ユニバースが小さすぎたこと」
は絶対リターンの悪化に確かに寄与していた（縮小改善という形で裏付けられた）
が、**それだけが敗因ではない**。225銘柄・先読みバイアスありの条件でも
なお勝てないことから、**Momentum/LowVol/Liquidityというファクター定義
自体、またはこの検証期間（2021-2026、TOPIXが+136%上げた一貫した強気
相場）がロング/ショート market-neutral戦略に不利だった可能性**が、
ユニバースサイズと並ぶ、あるいはそれ以上の要因として浮上した。

## ファクター定義・パラメータのグリッドサーチ + DSR/PBO検証（`factor_grid_analysis.py`）

225銘柄でも絶対リターンが正にならなかったことを受け、「パラメータの
選び方次第で勝ち筋がないか」を調べた。momentum_lookback(60/120/252日)×
top_q(0.1/0.2/0.3)×rebalance(月次/週次) の**全18通り**をグリッドサーチし、
`../backtest-framework/stats_eval.py` の Deflated Sharpe Ratio・PBOで
「グリッド内最良の結果が本物か、単なる多重検定の偶然か」を判定した。

```
最良パラメータ: momentum_lookback=252  top_q=0.1  rebalance=ME
  観測Sharpe(年率換算)=0.333  試行数=18
  期待される最大Sharpe(偶然・年率換算)=0.645
  Deflated Sharpe Ratio(DSR) = 0.247
  PBO(過学習確率) = 0.032
判定: DSR・PBOの基準を満たさない → 多重検定による偶然の可能性が高い
```

**グリッド内最良（252日モメンタム・上下10%・月次リバランス、Sharpe
+0.33）ですら、18通り試せば偶然に出てくると期待される最大Sharpe
（+0.645）を下回っていた。** つまりこの好成績は「本当に良いパラメータを
見つけた」のではなく、**18回試したことで生じた選択バイアスの範囲内**
という判定になる。

PBOが低い（0.032）点は一見矛盾するように見えるが、PBOは「IS最良が
OOSで中央値以下に落ちる頻度」を測るもので、DSRとは問いが異なる
（「このパラメータがIS/OOS間で相対的に安定して上位か」であって、
「絶対的に良い」かではない）。**両指標は補完的であり、DSRが低い時点で
「グリッド内最良を信頼できるα」として採用すべきではない**、という
判断を優先した。

## 複数レジームでの検証（15年・2011-2026、`--range 15y`）

単一の強気相場（2021-2026）だけでは評価が偏るため、2011年（東日本大震災後）
から現在までの15年、Abenomicsの大相場・2015-16年の停滞・2018年の下落・
2020年のコロナショックと急回復・2022年の調整・直近の強気相場と、複数の
レジームを含む期間で再検証した。

⚠️ **この検証中に別のデータ品質バグを発見・修正した**: `1306.T`
（TOPIX連動ETF）は2015年1月に10:1の株式分割を行っているが、Yahoo
FinanceのadjcloseはこれをRETROACTIVEに調整しておらず、
2014-12-30(986円)→2015-01-05(98円)という**偽の90%急落**が15年データに
混入していた（5年データではこの分割日を含まないため露見していなかった）。
`panel.py`に`_fix_unadjusted_splits`を追加し、持続的なレベルシフト
（一時的なグリッチとは異なり新水準がその後も続く場合のみ）を検出して
遡及調整するようにした。実装中、検出ロジックの順序を誤ると、別の
一時的な2日グリッチ（2026年3月30-31日、翌日には元の水準に戻る）の
「回復の一歩」を誤って「本物の分割」と誤認し、15年分の履歴全体を
誤って10倍にスケーリングしてしまう副作用も発見し、「先に1日グリッチ
除去→後で持続的な分割検出」の順に直して修正した。

修正後の実測結果:

## マクロ要因（為替感応度）ファクターの追加検証（`fx_sensitivity_analysis.py`）

ユーザーから「円安による日経平均上昇やコロナ禍のような時代背景・
マクロ要因が反映されていないのでは」という指摘を受けた。Momentum/
LowVol/Liquidityは価格・出来高の統計パターンのみで、為替・金利・
政策等のマクロ要因を一切考慮していない、という指摘は正しい。

そこで、**銘柄ごとのドル円変化への感応度（FXベータ）**を新しい
クロスセクショナル・ファクターとして構築し、既存ファクターと同じ
DSR/PBO基準で検証した。ドル円そのものは全銘柄共通の値なので横断比較の
材料にならないが、「この銘柄は円安にどれだけ反応するか」という
**銘柄ごとの感応度**は横断的なランキング材料になりうる、という設計。

### 検証結果

① **FXベータ単独のグリッドサーチ**（window×rebalance、6通り）:
最良（504日窓・月次リバランス）で総リターン+73.8%・Sharpe+0.371。
**DSR=0.714・PBO=0.016**——既存3ファクターのグリッドサーチ（最良でも
DSR=0.247）より明確に良好な結果だった。

② **既存3ファクターへFXを混ぜた場合の配分比率感度**:

| FX比率 | 総リターン | Sharpe |
|---|---|---|
| 0%（既存3ファクターのみ） | -48.7% | -0.366 |
| 25% | -27.9% | -0.172 |
| 50% | +28.6% | +0.217 |
| 75% | +60.8% | +0.337 |
| 90% | +67.5% | +0.352 |
| 100%（FXのみ） | +73.8% | +0.371 |

FX比率を上げるほど**単調に改善**し、最良は「既存3ファクターを使わず
FXのみ」だった。これは既存のMomentum/LowVol/Liquidity複合が、この
15年間では一貫して足を引っ張っていたことの裏返しでもある。

### 誠実な留保（過大評価しない）

- この配分比率グリッド自体でDSR/PBOを取り直すと**DSR=0.442・
  PBO=0.198**と、①ほどの確信度は出ない。同じ「最良」でも、どの試行
  集合と比較するか（window/rebalanceの6通り vs 配分比率の6通り）で
  DSRの値そのものが変わる——これはDSR/PBOという手法自体の限界であり、
  「多重検定を補正すれば絶対的な答えが出る」わけではないことを示す
- **最大の留保**: 検証期間（2011-2026）のドル円は78円→163円までほぼ
  一貫して円安方向に動いた**単一のFXスーパーサイクル**であり、複数の
  独立した為替サイクル（円高局面を含む）を経ていない。「FXベータ」が
  本当に構造的なリスクプレミアムなのか、単に「たまたまこの15年持続
  した一方向トレンドに乗れただけ」なのかを、この検証だけでは区別
  できない。円高局面が今後続いた場合に同じ関係が成り立つ保証はない
- ユーザーの指摘は正しく、マクロ要因を一つ加えるだけで既存ファクター
  単独よりは明確に良い結果が得られた。ただし「精度が上がるか」という
  問いへの答えは「単純に足すだけなら上がるとは限らないが、既存の
  弱いファクターを一部置き換える形なら改善余地がある。ただし単一の
  歴史的トレンドへの過学習の疑いは残る」という条件付きのものになる

## ループ学習型のファクター配分（`factor_rotation_walkforward.py`）

ユーザーから「FXベータをループ学習する分析ファクターの一つとして
活用すれば良い」という提案を受け、固定配分ではなく**各時点までの
データだけで直近成績を評価し、ファクター配分比率を毎期更新しながら
ウォークフォワードで運用する**仕組みを実装した。

設計: 候補ファクター{Momentum, LowVol, Liquidity, FX}について、年次の
評価日ごとに直近504日の実現Sharpeを計算し（**評価日より後のデータのみ
参照**）、直近Sharpeがプラスのファクターだけをその値に比例した重みで
組み合わせ、次の評価日まで固定して運用する。

### 実装時に発見・修正した2件のバグ

1. **先読みバグ**: 評価日tまでのデータで決めた重みを、tで終わる**同じ
   期間**に適用してしまっていた（本来はtより後、次の評価日までに
   適用すべき）。その年自身のリターンが重み決定に混入する典型的な
   先読みバイアスで、修正前は「ループ学習型」が異常に良い数値
   （Sharpe+0.07、ほぼ横ばい）を示していたが、修正後は明確に悪化した。
2. **「現金化」のつもりが実現していなかったバグ**: 全ファクターの直近
   Sharpeがマイナスの期間は重みを全て0にして現金化するつもりだったが、
   `strategy.backtest`はクロスセクショナル・ランキング方式（`nlargest`/
   `nsmallest`）のため、複合スコアが全銘柄で同一値(0)になると同点の中
   から任意選択してしまい、実質ランダムなポジションを取ってしまう
   （「ノーポジション」を表現できない）。等配分へのフォールバックに
   変更して修正した。

### 実測結果（両バグ修正後）

| 手法 | 総リターン | Sharpe |
|---|---|---|
| ループ学習型（年次動的配分） | -25.8% | -0.13 |
| 固定3ファクターのみ | -55.6% | -0.41 |
| 固定FXのみ | **+73.8%** | **+0.37** |
| （参考）ユニバース均等保有 | +620%（7.20倍） | - |

**ループ学習型は固定3ファクターより明確に改善したが、単純に「FXのみに
100%配分」した場合を上回れなかった。** 年次の配分比率を見ると、
2018年（Momentumの直近Sharpeが良く見えたため100%配分→実際の2018年は
Momentumが機能せず損失）や2019年（同様にLiquidityへ100%配分→やはり
不発）のように、**「直近良かったファクターに乗り換えた途端、その
ファクターの効き目が終わっていた」という典型的な「ファクター・
タイミングの罠」が繰り返し観察された**。これは学術的にもよく知られた
現象（トレンドフォロー的な戦略選択は、選んだ瞬間に反転しやすい）で、
今回の単純な「直近Sharpe比例配分」というループ学習の設計では、この罠を
回避できなかった。

### 改善案の追加検証: シュリンク型・リスクパリティ型

「より洗練された設計であれば改善するかもしれない」という上記の推測を
実際に検証するため、v1（チェイシング型）に加えて2つの改良版を実装した。

- **v2 シュリンク+下限型**: 目標配分を「前回配分と等配分の平均」方向へ
  50%シュリンクし、各ファクターに最低10%の配分を強制する。急激な
  乗り換え・単一ファクターへの過度な集中を抑える設計。
- **v3 リスクパリティ型**: リターンの良し悪しを一切見ず、各ファクターの
  直近ボラティリティの逆数だけで配分する（Dalio流のリスクパリティを
  銘柄間でなくファクター間に適用したもの）。

| 手法 | 総リターン | Sharpe |
|---|---|---|
| v1 チェイシング型（直近Sharpe比例） | -25.8% | -0.13 |
| v2 シュリンク+下限型 | -23.4% | -0.11 |
| v3 リスクパリティ型（ボラの逆数） | -48.3% | -0.43 |
| 固定3ファクターのみ | -55.6% | -0.41 |
| 固定FXのみ | **+73.8%** | **+0.37** |

DSR/PBOをこの5手法で比較しても、最良は依然「固定FXのみ」
（DSR=0.477・PBO=0.008）だった。

**分かったこと**: v2（乗り換えを緩やかにする）はv1よりわずかに改善した
（-25.8%→-23.4%）——「チェイシングの罠を和らげる」という発想の方向性
自体は正しかった。一方でv3（リターンを無視しボラだけで配分する
リスクパリティ型）は明確に悪化した（-48.3%）。これは、**FXが優れて
いた理由は「ボラが低いから」ではなく「リターンが良かったから」**であり、
リターンを一切見ないリスクパリティは、FXの優位性を全く捉えられず、
結局4ファクターへほぼ均等配分（各20〜25%）になってしまうため。「良い
ファクターを探す」問題に対して「リスクだけを見る」設計は原理的に
ミスマッチだった、という明確な教訓が得られた。

### 結論（過大評価しない）

ユーザーの提案自体は方向性として正しく、動的配分は固定の弱いファクター
構成よりは改善する。シュリンク・下限を加えるとチェイシングの罠は
多少和らぐが、効果は小さい。リスクベースの配分はこの問題設定には
そもそも不向きだった。**3つの動的配分手法のいずれも、静的に「最良の
単一ファクター（FX）を使い続ける」という最も単純な判断を上回れなかった**。
「ループ学習を使えば自動的に良くなる」わけではなく、その設計次第——
かつ今回検証した設計の範囲では、固定配分に対する明確な優位は見つから
なかった、というのが正直な結論。

## 材料（ファクター）を増やす: 金利感応度ファクターの追加（`rate_factor.py`）

配分ロジックの改良では埋まらなかったギャップに対し、「配分アルゴリズム
より先に、材料（ファクター）自体を増やす・良くすることの方が効くはず」
という仮説を検証した。FXベータと同じ発想で、**銘柄ごとの金利変化への
感応度（金利ベータ）**を新しいファクターとして追加した。

⚠️ 日本国債（JGB）の利回り系列はYahoo無料APIで15年分安定して取得
できない（JGB ETF `2510.T`は2017年以降のデータしかない）ため、**米国
10年国債利回り(`^TNX`)**で代用した。米金利は日米金利差を通じてドル円に
も影響するため、FXベータとの相関を確認したところ**横断相関の平均は
0.491**——完全な言い換えではないが、無関係でもない、部分的に独立した
情報だと確認した。

### 実測結果

FXベータ単独のグリッドサーチ同様、金利ベータ単独でも良好な結果:
最良（252日窓・月次）で総リターン+67.9%・Sharpe+0.40・**DSR=0.674・
PBO=0.091**——FXベータ（DSR=0.714）に匹敵する強さだった。

この金利ベータを5つ目の候補として動的配分（v1/v2/v3）に加えて
再検証したところ、**ループ学習型の成績が劇的に改善した**:

| 手法 | 総リターン（4ファクター時） | 総リターン（Rate追加後） |
|---|---|---|
| v1 チェイシング型 | -25.8% | **+26.9%**（Sharpe+0.20） |
| v2 シュリンク+下限型 | -23.4% | **+21.9%**（Sharpe+0.18） |
| v3 リスクパリティ型 | -48.3% | -7.2%（Sharpe-0.01） |
| （参考）固定FXのみ | +73.8% | +73.8% |
| （参考）固定Rateのみ | - | +67.9% |
| （参考）固定FX+Rate 50/50 | - | +40.2% |

**同じ配分アルゴリズムのまま、材料を4つから5つ（うち「本当に効く」
ものが1つから2つ）に増やしただけで、v1は-25.8%→+26.9%、v2は-23.4%→
+21.9%と、初めてプラスに転じた。** これは「配分ロジックを工夫する」
より「材料自体を増やす」方が遥かに効果が大きいという、ユーザーの
最優先事項の判断が正しかったことを実データで裏付ける結果である。

ただし、それでもなお**動的配分は「固定FXのみ」（+73.8%）や「固定
Rateのみ」（+67.9%）という最も単純な静的判断には届いていない**。
興味深いことに、FX+Rateを静的に50/50で混ぜても（+40.2%）単独使用
より劣っており、これは2つのファクターの相関が0.491程度では、単純な
均等配分では「良い方だけに集中する」ほどの効果を得られないことを
示している。DSR/PBOでの7手法比較でも最良は依然「固定Rateのみ」
（DSR=0.521・PBO=0.302）だった。

## 「材料を増やせば勝つ」の検証: 相関・PCA・クラスタリング（`factor_independence_analysis.py`）

ユーザーから重要な指摘を受けた: 「材料を増やせば改善する」という
言い方は不正確で、正確には「**独立した情報源を持つファクターを追加
すると改善する**」。FXベータ・金利ベータはどちらも「円安→米金利上昇→
輸出株」という**同じマクロドライバー**を見ている可能性があり、単純な
相関係数（スコアの横断相関0.491）だけでは実態を捉えきれない、という
指摘。これを実際に相関行列・PCA・階層クラスタリングで検証した。

### 実測結果（5ファクターの実現リターン系列、日次）

相関行列（実現リターンの相関。スコアの横断相関0.491より高い0.669が
FX-Rate間で出た——シグナル自体の相関より、実際にそれで売買した結果の
相関の方が強く出ることが分かった）:

|  | Momentum | LowVol | Liquidity | FX | Rate |
|---|---|---|---|---|---|
| Momentum | 1.00 | 0.03 | 0.32 | -0.08 | -0.18 |
| LowVol | 0.03 | 1.00 | -0.27 | **-0.50** | **-0.50** |
| Liquidity | 0.32 | -0.27 | 1.00 | -0.06 | -0.07 |
| FX | -0.08 | -0.50 | -0.06 | 1.00 | **0.67** |
| Rate | -0.18 | -0.50 | -0.07 | 0.67 | 1.00 |

PCA（主成分ごとの寄与率）: PC1=42.8%・PC2=27.8%・PC3=14.7%・PC4=8.3%・
PC5=6.4%。累積90%に達するには4主成分必要——固有値がなだらかに減衰
しており、単純な閾値だけを見ると「5個ともほぼ独立」に見えてしまう。

⚠️ **しかしこれは早計な結論だった**。階層クラスタリング（相関距離
1-|相関|）の結合順を見ると、実態がより明確になる:

```
結合1: FX + Rate            距離=0.331（最初に結合＝最も近い）
結合2: LowVol + (FX+Rate)   距離=0.503（LowVolはFX/Rateの「裏返し」）
結合3: Momentum + Liquidity 距離=0.685（別の独立したクラスタ）
結合4: 上記2クラスタが統合  距離=0.887（ここでようやく合流）
```

**FXとRateは最も強く結合し、LowVolはその「符号反転版」として同じ
クラスタに実質的に含まれる**（円安・金利上昇というリスクオン環境で
低ボラ・ディフェンシブ銘柄が相対的に劣後する、という経済的に筋の
通った関係）。つまり実質的には**2つの大きなテーマ**しかない:

- **テーマA（マクロ/リスクオン軸）**: FX・Rate・（符号反転）LowVol
- **テーマB（価格テクニカル軸）**: Momentum・Liquidity

単純な相関係数の閾値だけで「独立か」を判断すると、符号が逆の関係
（LowVolのケース）を見逃す。PCAの寄与率だけでも、固有値がなだらかに
減衰する場合は「実質何個の軸があるか」を過大評価しやすい。**クラスタ
リングまで見て初めて「実質2テーマ」という構造が見えた**、というのが
重要な発見だった。

### 含意: 次に何を追加すべきか

この結果は、ユーザーが懸念した通り、**コモディティ感応度・ドル指数
感応度など「別のマクロ感応度」をさらに追加しても、おそらくテーマA
（マクロ/リスクオン軸）を再び測るだけで、真の分散効果は限定的**である
ことを示唆する。次に探すべきは、テーマA・テーマBのどちらとも異なる
**第3の軸**を捉えるファクターであるべき。

ユーザーが提案した候補（信用スプレッド・EPSリビジョン・アナリスト
予想変化・財務健全性・バリュエーション再評価）はいずれも有望な方向だが、
⚠️ **これらの多くは本シリーズが繰り返し直面してきたのと同じデータ制約
に直面する**: Yahoo無料APIは現在のスナップショットしか提供せず、
15年分の点in-time履歴（過去のある時点でアナリスト予想がどうだったか、
信用スプレッドがどうだったか）は取得できない。過去バックテストへ
現在のスナップショットを一律適用すれば先読みバイアスになる（本
シリーズの`buffett_style.py`等で繰り返し明記してきた制約と同じ）。

価格データだけで先読みなく構築でき、かつテーマA・Bのどちらとも異なる
可能性がある候補としては、**短期リバーサル**（1ヶ月リターンの逆、
12ヶ月モメンタムとは学術的に異なる軸として知られる）や、**アイディオ
シンクラティック・ボラティリティ**（市場・ベータ除去後の残差ボラ）
などが、次に試す価値のある現実的な選択肢である。

## 第3の軸の候補を3段階ゲートで検証（`reversal_ivol_factor.py`）

ユーザーが提案した3段階ゲート方式で、短期リバーサル・IVOLの2候補を
検証した: **①新規性**（既存ファクターとの相関・クラスタリングで
確認）→**②単独の統計的評価**（DSR/PBO）→**③限界貢献度**（ポート
フォリオに追加した際の改善）。後段のゲートで落ちれば、それ以降は
検証しない、という順序で進めた。

### 短期リバーサル（1ヶ月リターンの逆）

②単独のグリッドサーチ（window×rebalance）で**全組み合わせがマイナス
Sharpe**（最良でも-0.192）、DSR=0.017・PBO=0.171——**明確に不合格**。
①の相関を確認すると理由が分かる: **Momentumとの相関が-0.851**という
極めて強い負の相関だった。これは学術的に「別物」とされる12ヶ月
モメンタムと1ヶ月リバーサルが、この検証設定（同じ価格系列・似た
ホライズン）では実質的に「符号を反転させただけのテーマB」になって
いたことを意味する。**新規性ゲートの時点で不合格**であり、③限界
貢献度の検証は行わなかった。

### アイディオシンクラティック・ボラティリティ（IVOL）

②では興味深い結果が出た: 「低IVOL選好」（Ang et al. 2006の低IVOL
アノマリー通りの符号）は全窓でマイナスSharpeだったが、**「高IVOL
選好」（学術的な定説とは逆の符号）で最良（120日窓）Sharpe+0.573・
総リターン+106.9%**という強い数字が出た。

⚠️ しかしDSR=0.063と極めて低く、**統計的には不合格**と判断した。
さらに重要な方法論上の注意点: この検証は「符号（高IVOL好き/低IVOL
好き）」自体を事後的に探索しており、これは通常のDSR計算が想定する
「同じ方向性の中でのパラメータ探索」よりも危険なデータスヌーピング
（"符号フィッシング"）である。DSRの数値をそのまま信じるのではなく、
**むしろ実際の信頼度はこの数値よりさらに低いと考えるべき**。

①の相関を見ると、これも新規性が低いと分かる: **LowVolとの相関が
-0.650**——「高IVOL選好」が良かったのは、既存のLowVol（-0.90という
本シリーズ最悪のSharpeだった）の「符号を反転させただけ」という
解釈と整合的。さらにFX(0.257)・Rate(0.353)とも無視できない相関があり、
テーマAの一部も同時に測っている。**単独ではテーマAの劣化コピー＋
既存の失敗ファクターの裏返し**であり、これも新規性ゲートで実質不合格。

### 結論（過大評価しない）

**ユーザー提案の3段階ゲートは狙い通り機能した。** 短期リバーサル・
IVOLはいずれも、学術的には「別の現象」とされていながら、この日本株
15年間の検証データでは、既存のテーマA・テーマBの言い換え（符号
反転）に過ぎないと判明し、どちらも採用を見送った。「第3の軸」は
まだ見つかっていない。次に探すべき方向性（信用スプレッド・EPS
リビジョン等）は、いずれもYahoo無料APIの点in-time制約に阻まれる
可能性が高く、この壁を越えるにはJ-Quants等の別データソースの統合が
必要になる、というのが現時点の到達点。

## 今回の到達点と正直な結論

- **設計を実現できた**: β≈0の市場ニュートラルL/Sを、コスト・空売り・分割調整・
  異常値除去まで含めて実装。さらに**Yahoo財務でQualityを取得し、4ファクター版の
  ライブランキング（設計の完全形）を実現**した
- **ユニバース拡大（25→225）で絶対リターンは大幅改善したが、依然マイナス**。
  「ユニバースが小さすぎた」という仮説は部分的に正しかったが、それだけが
  敗因ではなかった
- **含意**: J-Quantsで点in-time Qualityを苦労して統合する前に、ファクター定義
  自体（Momentum/LowVol/Liquidityの重み・定義）や、より長期・複数レジームを

  （Abenomics相場・停滞・弱気・コロナショック・調整・直近強気）で
  再検証したが、15年中11年がマイナスで、β≈0の市場ニュートラル性は
  レジームを問わず維持される一方、絶対リターンはレジームに関わらず
  概ねマイナスだった。ただし直近3年（2024-2026）だけは強気相場でも
  プラスという観察はあり、断定はできないが記録した

  生存者バイアスもあり、TOPIX500が持つ中小型株のクロスセクションを欠く）
- 財務データ非対応のため Quality は欠落（設計の中核が未実装）
- 日足・終値ベースの簡易執行モデル。実運用のスリッページ・板の薄さは未反映
- 15年（2011-2026）の複数レジームで検証済みだが、日経225の225銘柄という
  同一ユニバース・同一ファクター定義のままなので、ファクター設計自体の
  限界までは切り分けられていない
- Yahoo Financeのadjcloseは分割・併合を必ず遡及調整しているとは限らない
  （実例: `1306.T`の2015年10:1分割）。`_fix_unadjusted_splits`で持続的な
  レベルシフトのみ検出・修正しているが、検出漏れ・誤検知のリスクは
  完全には排除できない
- **このツールは検証のみ**。実発注機能は持たない

## 研究フェーズの結論（価格ベース探索の終了基準）

ここまでの一連の検証（ユニバース拡大→パラメータのグリッドサーチ→
複数レジーム検証→FX/Rateベータの追加→ループ学習→相関/PCA/クラスタ
リング→第3軸候補の3段階ゲート）を経て、**「価格・出来高だけからの
ファクター探索」という研究の一区切り**として、以下を明示する。

### 分かったこと

**価格・出来高だけから構築可能な代表的特徴量（Momentum・LowVol・
Liquidity・短期リバーサル・アイディオシンクラティック・ボラティリティ）
を、日本株15年データ・225銘柄で比較検証した結果、これらは統計的に
実質2つの潜在情報軸へ集約された**:

- テーマA（マクロ/リスクオン軸）: FX感応度・金利感応度・（符号反転）LowVol
- テーマB（価格テクニカル軸）: Momentum・Liquidity

新規候補として追加した短期リバーサル・IVOLも、この2軸の変形（それぞれ
Momentumとの相関-0.851、LowVolとの相関-0.650）として説明でき、
**統計的に独立した第3軸はこの検証範囲では確認できなかった**。

### 追記: 「試していない候補が多いのに結論してよいのか」への再検証

一度この結論で研究を凍結した後、ユーザーから「悪魔の代弁者」として
「試していないものがたくさんあるのに、これで研究を締めていいのか」
という再反論を受けた。直近に不合格となったReversal・IVOLは、実は
**数学的に既存ファクターと近い性質を持っていた**という自己批判
（Reversal=1次モーメントの符号違い、IVOL=2次モーメントの残差版）を
踏まえ、**数学的に真に独立した性質を持つ候補**を2つ追加で検証した:

- **歪度（`skewness_factor.py`）**: リターン分布の3次モーメント。
  1次（平均・トレンド）・2次（分散・共分散）とは数学的に独立。
  Bali, Cakici & Whitelaw (2011)の「宝くじ効果」に基づき、低歪度選好の
  符号で単独グリッドサーチ（window×rebalance）を実施したが、**全組み合わせが
  マイナスSharpe**（最良でもSharpe-0.421、DSR=0.0）。逆符号（高歪度選好、
  参考値のみ）も試したが同様にマイナス（Sharpe-0.78）。**符号によらず
  明確に不合格**（IVOLのような符号フィッシングの余地がない、クリーンな
  否定結果）。
- **共動性（`comovement_factor.py`）**: ユニバース平均リターンとの
  ローリング相関。分散・共分散とは別角度の「連動度」を狙ったが、
  グリッドサーチ全8通り（window×rebalance×符号）がマイナスSharpe
  （最良でもSharpe-0.185）、DSR=0.099・PBO=0.873（極めて高い過学習
  懸念）で明確に不合格。

**①新規性ゲート（既存7ファクターとの相関）で興味深い非対称性が
出た**:

|  | Momentum | LowVol | Liquidity | FX | Rate |
|---|---|---|---|---|---|
| **歪度** | -0.29 | 0.07 | -0.13 | -0.04 | -0.05 |
| **共動性** | -0.15 | -0.54 | -0.05 | **0.66** | **0.61** |

**歪度は既存7ファクターすべてと相関が0.3未満——このシリーズで検証した
中で最も「新規性の高い」候補だった。** 一方、共動性はFX(0.66)・
Rate(0.61)と強く相関し、**結局テーマAの別表現に過ぎない**と判明した
（市場との連動度が高い銘柄ほど、円安・金利上昇というリスクオン環境で
有利という、経済的に筋の通った関係）。

**これは当初の結論を弱めるのではなく、むしろ補強する形になった**:
歪度という真に独立した情報軸は実在したが、**統計的に独立であることと
リターンを予測する力を持つことは別問題**であり、歪度はこの15年・
225銘柄という検証範囲ではリターンの予測力を持たなかった。共動性は
そもそも独立ですらなかった。「第3の軸を探したが、真に独立したものは
見つかったが、それは無意味（無相関だが無力）だった」という、より
解像度の高い結論に至った。

### 終了基準（この研究の到達点と限界の明示）

**本研究の範囲（価格・出来高のみ、無料で取得可能な履歴データ、日本株
15年）では、独立した第3の情報軸を確認できなかった。今後の改善余地は
アルゴリズムの高度化ではなく、Point-in-Time財務情報やEPSリビジョン等、
価格とは独立した情報源の導入にある。**

これは「限界があった」というだけでなく、**「どこまで分かり、どこから
先はデータの問題なのか」を明確に切り分けた**という研究上の到達点である。
今回、パラメータチューニング（グリッドサーチ+DSR/PBO）・検証期間の
拡張（単一レジームの疑いを15年複数レジームで解消）・配分アルゴリズムの
高度化（ループ学習3種）・特徴量エンジニアリング（FX/Rate/リバーサル/
IVOL）と、モデル側で試せることは一通り試した上でこの結論に至っており、
**現在のボトルネックはモデルではなくデータである**、と言える状態まで
検証を積み上げた。

### このフェーズの本当の成果は何だったか

Sharpe比の数値そのものより価値があるのは、**候補評価パイプラインが
完成したこと**である:

```
候補 → 単独DSR → PBO → 既存ファクターとの相関 → PCA → クラスタリング
     → （ポートフォリオへの）限界貢献度 → 採用/不採用
```

FX・Rateベータで改善が見えた際も、この評価パイプラインを最後まで
通し、「単一のFXスーパーサイクルへの依存かもしれない」「既存テーマの
再測定に過ぎないかもしれない」という留保を最後まで残した。「価格
データだけでは独立した第3軸を確認できなかった」という結論に説得力が
あるのは、うまくいったと見えた時点で結論を急がず、この評価パイプ
ラインを最後まで一貫して適用したからである。

### 次のフェーズは「続き」ではなく「別プロジェクト」

このフェーズ（**Project A: 価格・出来高だけで独立したアルファ・情報軸
を作れるか**）と、次に取り組みうるPoint-in-Timeデータ統合
（**Project B: Point-in-Time情報を加えると本当に新しい情報軸が
得られるか**）は、**問い自体が異なる**。混同せず、別プロジェクトとして
扱うべき、というのがこのフェーズを終えての判断である。

Project Bに着手する際も、いきなり大量のファクターを作るのではなく、
まず（例えば）Point-in-TimeのEPS修正率・ROE等から1つの候補を作り、
既存の価格ベース2軸（マクロ/リスクオン軸・価格テクニカル軸）との
相関・クラスタリングを確認し、統計的に独立した情報源だと確かめて
から初めてバックテストへ進む、という順序を踏むべきである。本フェーズ
で確立した検証パイプライン（backtest-framework、DSR/PBO、15年複数
レジーム評価、相関/PCA/クラスタリング）は、そのままProject Bでも
同じ検証器として再利用する。それによって「データを変えたことによる
進歩」と「モデルをいじっただけの変化」を明確に切り分けられる。
