## 本文

AIエージェントに「タスクをやらせる」のと「役職を任せる」のは、まったく
別物だと気づいたところから、この話は始まります。

Claude Codeで副業の自動化を試そうとしたとき、最初にぶつかった壁は
単純でした。AIに「市場調査して」「これ売って」と頼んでも、毎回やり方が
ブレる。良い時もあれば、適当な時もある。これでは仕組みとして使えません。

そこで発想を変えました。AIに「タスク」を渡すのではなく、「役職」を
渡すことにしたのです。

### CEO・PM・7部門をAIエージェントにする

やったことはシンプルです。CEO、PM（COO）、市場調査部、検証部、事業評価部、
制作部、マーケティング部……それぞれの役職に「契約書」を持たせました。

- Mission（何のためにいるか）
- Inputs / Outputs（何を受け取り、何を出すか）
- Cannot do（やってはいけないこと）
- Exit Criteria（何をもって仕事終わりとするか）

これを1つずつ文書化して、Claude Codeに読ませる。すると、案件が来る
たびに同じ手順・同じ品質でAIが動くようになりました。

流れはこうです。

```
市場調査 → 検証（品質監査） → 事業評価（投資判断） → CEO承認
→ 制作 → マーケティング → （分析・改善）
```

ポイントは「検証」を挟んだことです。市場調査AIが集めてきた情報を、
そのまま信じない。別のAIが「このエビデンス、URLちゃんとある？」
「情報古くない？」と監査してから、初めて評価・投資判断に進みます。
AIの自己申告を鵜呑みにしない仕組み、地味ですが一番効いています。

### 「完成」と「完了」は違う

作る側の設計はわりとすぐ形になりました。むしろ苦労したのは、
実際に世に出す段階です。

商品を3つ作り終えた時点では、正直「もう終わった」気になっていました。
でも実際にはBOOTHに1つも出品していない。ここで気づいたのは、
「完成」と「完了」はまったく別物だということです。

会社のルールに「公開して、初めて売上が発生して、そこでようやく完了」
という一文を追加しました。作ることに満足して、世に出す一歩を後回しに
しないための、自分への戒めのようなものです。

### 数字で判断する、盛らない

もう一つ大事にしているのは、Evidenceに基づかない判断をしないことです。
市場調査の結果、実は「公式統計」と呼べるような一次情報はほとんど
見つかりません。個人ブログやレビュー記事が中心になることも多い。
だから、自信度（Confidence Score）は高く盛らず、70〜80%程度に
とどめています。

利益が見込めそうでも、品質や信頼を犠牲にする提案は通さない、という
ルールも作りました。売上のためだけの誇張表現は書かない。地味ですが、
これが一番長く続けられる方法だと思っています。

### 今のところの実績（正直に）

- 商品3点（Claude Code用のテンプレート・コマンド集）を制作
- BOOTHショップを開設し、3商品とも公開
- Xフォロワーは77人（始めたばかり）
- 売上はまだ0件

派手な数字は何もありません。ただ、「作る→検証する→世に出す→振り返る」
という一連の流れを、実際に手を動かして1週間で回せたことは、自分の中では
それなりに手応えがあります。

引き続き、このプロセスをそのまま記録として公開していこうと思います。
