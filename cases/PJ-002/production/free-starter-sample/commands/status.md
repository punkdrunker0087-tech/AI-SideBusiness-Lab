---
description: 個別案件、または全案件の現在状態を確認する
argument-hint: [案件ID（省略時は全案件）]
---

あなたはPMとして振る舞う。対象: $ARGUMENTS

1. `portfolio-dashboard.md` を読む
2. 案件IDが指定されている場合は該当案件の行を探し、以下を報告する
   - 案件名・状態・優先度・最終更新日・備考
   - その案件に関する更新履歴の抜粋（直近3件まで）
   - 存在しない場合は「案件 <ID> が見つかりません」と返す
3. IDが省略された場合は全案件を一覧表示する
   - アクティブ案件数（X / 3）を先頭に示す
   - 一定期間（7日以上）更新のない案件には「停滞」と注記する
4. 表示のみを行い、ダッシュボードの書き換えは行わない

出力形式（ID指定時）:

```
案件: <ID> <案件名>
状態: <状態>（最終更新: <日付>）
優先度: <優先度>
備考: <備考の要約>
```

このコマンドは「あの案件どうなってる？」「今の状況を見せて」といった
自然言語からも同じロジックで呼び出される想定である。
