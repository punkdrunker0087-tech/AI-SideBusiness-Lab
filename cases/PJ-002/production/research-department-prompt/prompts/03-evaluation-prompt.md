# 03. Evaluation Prompt — 8軸スコアリング投資判断AI

02のValidation PromptでPASSした調査結果を、8つの軸で採点し、Go/No-Go
を数値で判断させるプロンプトです。「勘」ではなく「ルール」で判断する
ための核となるプロンプトです。

---

## プロンプト本文

```
あなたは投資委員会として振る舞ってください。以下のPASS済み市場調査
結果をもとに、8つの軸をそれぞれ0〜100点で採点してください。市場調査
側のRecommendationやConfidence Scoreをそのまま信じず、独立して採点
してください。

[ここにPASS済みの市場調査結果を貼り付ける]

## 採点軸と重み

- 市場性（20%）: 市場規模・需要の大きさ
- 収益性（20%）: 見込める利益率・単価
- 競争優位性（15%）: 差別化のしやすさ、競合の少なさ・弱さ
- 開発容易性（10%）: 週末稼働・少人数で実現できるか
- AI自動化率（10%）: AIだけでどれだけ工程を回せるか
- リスク（10%）: 法務・運用・レピュテーションリスクの小ささ
  （高スコア＝低リスク）
- Evidence品質（10%）: 各EvidenceのランクA〜E（A=5,B=4,C=3,D=2,E=1）を
  使い、次の式で算出する。
  Evidence品質スコア = (Σ各Evidenceのランク重み) ÷ (Evidence件数×5) × 100
- 戦略適合性（5%）: あなた自身の目的・方針との整合性

総合スコア = Σ（各軸の点数 × 重み）

## 判定基準

- 85〜100点: Strong Go（即着手）
- 70〜84点: Go（着手候補）
- 55〜69点: Conditional Go（条件付き。改善案を出してから再評価）
- 40〜54点: Pivot（方向転換が必要）
- 0〜39点: No-Go（見送り）

以下の形式で出力してください。

案件: <案件名>
市場性 <点数> / 収益性 <点数> / 競争優位性 <点数> / 開発容易性 <点数> /
AI自動化率 <点数> / リスク <点数> / Evidence品質 <点数> / 戦略適合性 <点数>
総合: <点数>
判定: <Strong Go/Go/Conditional Go/Pivot/No-Go>
理由: <各軸の要点を数行で>

最終的な実行判断はあなた自身（人間）が行ってください。このAIは判定案を
提示するだけです。
```

---

## 使い方のコツ

- 複数の案件候補がある場合は、同じプロンプトで並べて採点すると比較が
  しやすくなります
- 重みや閾値は「そのまま使う」もよし、自分の事業に合わせて調整しても
  構いません。ただし調整した場合は理由をメモに残しておくと、後で
  見返したときに判断の一貫性を保てます
