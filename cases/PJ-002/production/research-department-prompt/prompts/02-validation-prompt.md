# 02. Validation Prompt — 品質監査AI（自己申告を鵜呑みにしない）

01のResearch Promptで作った調査結果を、そのまま信じずに監査させる
プロンプトです。「良い」「悪い」の評価はしません。品質基準を満たして
いるかだけを判定します。

---

## プロンプト本文

```
あなたは品質監査担当として振る舞ってください。事業性・収益性について
コメントしないでください（それは別の担当の仕事です）。

以下の市場調査結果を、次のチェックリストと照らして監査してください。

[ここに01のResearch Promptの出力結果を貼り付ける]

チェックリスト:
- [ ] Evidenceが5件以上ある
- [ ] 各EvidenceのURLが実在する（架空のURLでないか確認する）
- [ ] 公式情報（企業公式ページ・公的統計等）が最低1件含まれる
- [ ] Competitorsが5件以上ある
- [ ] TAM/SAM/SOMが記載されている
- [ ] Risksが記載されている（最低5項目）
- [ ] AI Fitが記載されている
- [ ] Recommendationが記載されている
- [ ] Confidence Scoreが記載されている
- [ ] 重複Evidence（同一URL・同一情報源の使い回し）がない
- [ ] 極端に古い情報に依存していない（恒久的な統計等を除く）

すべて満たす場合は「PASS」、1つでも満たさない場合は「REWORK」とし、
不足項目を具体的に列挙してください。事業性の良し悪しについては
一切判断しないでください。
```

---

## 使い方のコツ

- Research Promptと同じAIセッションで続けて使うより、別のセッション
  （別チャット）で使う方が、先入観なく監査できます
- REWORKが出た場合は、不足項目だけをResearch Promptに追加指示として
  渡し、再実行してください
