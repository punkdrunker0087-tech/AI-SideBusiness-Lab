# マクロ投資 研究フレームワーク

グローバルマクロ運用（Bridgewater等）で重視される考え方を整理した研究資料。

> **スコープについて**: このドキュメントは学習・研究のための概念整理であり、
> 実運用できる体系的マクロ戦略・資産配分ロジック・レジーム自動検知エンジン・
> 売買ルールの実装は含まない。理由は、それらを組み合わせると実質的に
> 「高度な自動投資システム」の構築に直結してしまうため。ここでは代わりに、
> **何を・なぜ観察するか**という研究プロセスと、**過去に観測されてきた
> 傾向**（一次・二次情報に基づく）を整理する。このリポジトリの他ディレクトリ
> （`stat-arb/`・`multifactor-investing/`等）が「1つの市場・1つの手法」を
> 動くコードで検証したのに対し、本ドキュメントは「複数資産・複数マクロ環境」
> を横断する視点をコードなしで学ぶためのものと位置づける。

## マクロ研究の基本フレーム

「資産価格は経済環境の変化に反応する」を出発点に、次の順で分析する。

1. **経済環境の把握** — 成長・インフレ・金融政策の現状をデータで捉える
2. **市場が何を織り込んでいるかの確認** — 金利先物・イールドカーブ・
   インプライドボラティリティ等から市場の期待を読む
3. **資産ごとの感応度の分析** — 各資産クラスが経済環境の変化にどう反応
   してきたかを長期データで確認する
4. **ポートフォリオ全体のリスク評価** — 個別資産でなく、組み合わせた際の
   相関・分散効果を評価する
5. **仮説の継続的な検証** — 経済環境の定義や資産の反応パターンは固定的
   でなく、市場構造の変化に応じて絶えず検証し直す

## 監視対象マクロ指標

単一指標でなく、複数を組み合わせて評価する。

| カテゴリ | 代表的指標 | 更新頻度の目安 |
|---|---|---|
| 成長 | GDP成長率・小売売上高・製造業/サービス業景況感(PMI) | 四半期〜月次 |
| 物価 | CPI・PPI・賃金動向 | 月次 |
| 金融政策・金利 | 政策金利・長短金利・イールドカーブ | 随時〜月次 |
| 雇用 | 雇用統計・失業率 | 月次 |
| 企業 | 企業利益（決算） | 四半期 |
| 信用市場 | クレジットスプレッド | 日次 |
| 為替・資本フロー | 通貨・資本フロー統計 | 日次〜月次 |

更新頻度・速報性・改定（リビジョン）の有無が指標ごとに異なるため、
「最新の一点」だけでなく改定履歴も含めて評価するのが実務の作法とされる。

## 経済レジームの考え方（成長×インフレの4象限）

Bridgewater Associatesが体系化したことで知られる枠組みでは、**経済成長が
上向きか下向きか**・**インフレが上昇局面か下降局面か**の2軸で環境を
4つに区分する。

| | インフレ低下 | インフレ上昇 |
|---|---|---|
| **成長加速** | Goldilocks（適温） | Reflation（リフレ） |
| **成長鈍化** | Deflation（デフレ的） | Stagflation（スタグフレーション） |

Bridgewaterの「All Weather」ポートフォリオは、特定のレジームが最も
起こりやすいと賭けるのではなく、**4つの環境すべてにリスクを均等配分する**
という発想で設計されたことで知られる（1996年、Ray Dalioの信託資産向けに
開発）。

### 過去に観測されてきた資産クラスの傾向（一般論・研究report ベース）

以下は複数の市場調査・学術文献で報告されてきた**傾向**であり、
「常にこう動く」という法則ではない（後述の限界を参照）。

- **Goldilocks（成長加速・インフレ低下）**: 株式・国債ともに良好となりやすい
  局面として報告されることが多い。インフレ率が（水準の高低に関わらず）
  **減速している**局面が株式・国債双方にとって良好、という指摘がある
- **Reflation（成長加速・インフレ上昇）**: コモディティ・実物資産・
  資源国通貨にとって良好とされる局面。ある調査ではコモディティの平均
  リターンがこの局面で+25%程度という報告がある
- **Stagflation（成長鈍化・インフレ上昇）**: ほぼ全資産にとって厳しい局面
  とされ、現金・短期債が相対的な逃避先とされる。ある調査ではS&P500の
  平均リターンがこの局面で-8%程度という報告がある
- **Deflation（成長鈍化・インフレ低下）**: 需要主導の景気後退では国債
  など守りの資産が優位になりやすいとされる

**重要な留保**: これらは特定期間・特定市場の集計値であり、国・時代・
政策レジーム・市場構造によって関係は変化する。「常にこう動く」と
決めつけず、`stat-arb/regime.py`や`risk-management/market_risk.py`で
実装したような**継続的な検証・構造変化の監視**が研究の本質になる。

## 資産クラスの分析（着眼点）

| 資産クラス | 主な感応度ドライバー |
|---|---|
| 株式 | 成長（企業収益の見通し）への感応度が高い |
| 債券 | 実質金利への感応度が高い |
| コモディティ | インフレへの感応度が高い |
| 為替 | 金利差・資本フロー・相対成長率 |

これらの「主要ドライバー」は経験則として広く参照されるが、政策レジーム
（金融抑圧・量的緩和・為替介入等）によって関係の強さは変化しうる。

## リスク分散の考え方

複数資産を組み合わせる際、単純な資産クラス別配分だけでなく、次の軸を
多面的に見る。

- **相関**: 資産間の相関は時間とともに変化する（`stat-arb/`・
  `risk-management/`で実装した相関監視の考え方がそのまま応用できる）
- **ボラティリティ**: 資産ごとの変動幅の違いをリスク配分に反映する
- **流動性**: 危機時に売却できるかどうか
- **地域**: 特定国・地域への集中を避ける
- **通貨**: 為替エクスポージャーをヘッジするか、意図的に取るか

## 市場環境の変化への対応

危機時や政策転換時には、平常時と異なる振る舞いが観測されることが多い。

- **相関構造の変化**: 平常時に分散されている資産も、ストレス時には
  相関が上昇し「一斉に売られる」ことがある（`risk-management/market_risk.py`
  で実データにより実証した現象と同種）
- **ボラティリティの変化**: 危機時にはボラティリティが急上昇し、
  ポジションサイジングの前提が崩れる
- **流動性の変化**: 平常時に流動的だった資産が、ストレス時に極端に
  流動性を失うことがある（`risk-management/liquidity.py`の考え方と同種）

これらの変化を「通常時との違い」として定量的に捉えることが研究の目的。

## 地政学リスク

選挙・紛争・政策変更等のイベントは市場参加者の期待やリスク認識に影響する。

研究では、こうした不確実性を定量化する試みも行われている。代表例が
Federal ReserveのCaldara & Iacoviello (2022) による**Geopolitical Risk
(GPR) Index**で、主要新聞のアーカイブから地政学関連の記事頻度を集計し
指数化したもの（戦争の脅威・軍備増強・テロ等8カテゴリで分類）。この
指数の上昇は投資・雇用の減少や市場ストレスの増大と関連することが報告
されている。

研究アプローチとしては、

- **過去の類似事例**の収集（同種のイベントで市場がどう反応したか）
- **市場の反応**の定量化（イベント前後のボラティリティ・相関の変化）
- **シナリオ分析**（複数の展開を想定し、ポートフォリオへの影響を評価）

が一般的に用いられる。

## 研究で重視されること

個別の予測を当てることよりも、次の点が重視される。

- 経済環境をどのように定義するか（成長×インフレの4象限はその一例に過ぎない）
- その定義が異なる期間・地域でも一定の説明力を持つか
- 市場構造の変化（政策レジームの転換等）に対応できるか
- リスクを適切に管理できるか（個別資産でなくポートフォリオ全体で）

これは本リポジトリの他の研究フレームワーク（`backtest-framework/`の
Deflated Sharpe Ratio・PBO、`signal-research/`のIC評価、`stat-arb/`の
レジーム監視）と同じ精神——「良い結果が出た」ことより「頑健で再現性の
ある知見か」を慎重に評価する——をマクロ・クロスアセットの視点に
拡張したものである。

## このドキュメントでできないこと（意図的なスコープ外）

- 現在の経済環境を自動でレジーム分類するコード
- レジームに応じて資産配分を自動決定するロジック
- マクロシグナルに基づく自動売買ルール

これらを組み合わせた体系は、事実上の自動投資システムであり、本ドキュメント
の目的（考え方の理解）を超える。学習目的でコードを書きたい場合は、
`signal-research/`（単一シグナルのIC評価）や`risk-management/`
（相関・ボラティリティ監視）など、範囲を絞った既存フレームワークを
参照・拡張することを勧める。
