# LIMIT CONTENT（仮称） — AI運営のメディア会社OS

LIMIT LAB（LIMIT LAB WORKS）を引き継ぎながら展開する、**AI運営のメディア会社**。
動画は成果物であり、**商品は「AI会社を毎日運営した実録」**。実録をショート動画・
note・テンプレート・書籍・講座へ資産化する。法人登記はしない（実法人化は
`CHARTER.md`の発動条件を満たした時のみ検討する）。

## LIMIT LABとの関係

```
人間のCEO（同一人物）
 ├── LIMIT LAB WORKS（既存） … プロダクト販売の軸（BOOTH/note/Zenn）
 └── LIMIT CONTENT（本社）  … コンテンツ制作の軸（この会社）
```

- **憲法は共有**: LIMIT LABの`CONSTITUTION.md`（信頼と信用・先義後利・
  利用者第一・感動品質・技術は手段）をそのまま最上位規範として継承する。
  独自の憲法は作らない
- **知見も共有**: `knowledge/`（Decision Log・SOP・Institutional Memory）は
  グループ共通資産として参照・追記する
- **数字は分ける**: 売上・KPIはLIMIT LABと分けて記録し、会社ごとに
  Go/No-Go判断できるようにする

## ドキュメント構成

| ファイル | 内容 |
|---|---|
| `CHARTER.md` | 仮想定款。事業目的・引き継ぎ条項・実法人化の発動条件 |
| `ORG.md` | 組織設計。「AIが社長という設定」・アーカイブ部・実験番号・2系統KPI・量産の規律 |
| `EXPERIMENTS.md` | 実験台帳。全活動をEXP-NNNで採番し、成果物を紐付ける主キー |
| `pipeline/` | ショート動画 全自動生成パイプライン（SOP・テンプレート・実測済み） |
| `../docs/company-setup/` | 実法人化する場合の手続き・費用・AIにできないこと調査 |
