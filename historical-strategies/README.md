# 歴史的投資手法の要約と現代データへの翻訳（「歴史×AI」シリーズ）

「ウォーレン・バフェットのバリュー投資」「タートルズのトレンドフォロー
戦略」「Joel GreenblattのMagic Formula」という歴史的に有名な手法を正確に
要約し、現代の日本株データでどう再現・検証できるかを実装する。

> 研究・学習用。発注機能は持たない。

## このシリーズの研究プロセス（共通10段階）

各戦略について、以下の10段階を踏む。①〜⑤は各戦略モジュールのdocstring、
⑥以降は`pipeline.py`（共通処理）または各READMEセクションで扱う。

① 原典調査 → ② 論文調査 → ③ ルール抽出 → ④ 数式化 → ⑤ Python実装
→ ⑥ バックテスト → ⑦ 感度分析 → ⑧ レジーム分析
→ ⑨ どの市場で機能するか → ⑩ どこで破綻するか

同様に価値ある知見として扱い、過剰な一般化は避ける。**

## 1. タートルズのトレンドフォロー戦略

### 要約（出典付き）

1983年、商品先物トレーダーのRichard Dennisが、相棒William Eckhardtとの
「トレードは教えられるか」という賭けに端を発し、素人集団（"タートルズ"）に
厳格なルールベースの手法を教え込んだ。彼らは4年間で1億5000万ドル超の
利益を上げたとされる。

原典のルール（"The Original Turtle Trading Rules"）の核心:

- **N（ボラティリティ単位）**: True Rangeの20日指数平滑平均
- **System 1**: 20日高値ブレイクでエントリー、10日安値でエグジット
- **System 2**: 55日高値ブレイクでエントリー、20日安値でエグジット
- **ポジションサイズ**: 1ユニット = 資金の1%リスク ÷ N
- **ストップロス**: エントリーから2N
- **ピラミッディング**: 0.5N有利に動くごとに1ユニット追加、最大4ユニット
- **whipsawフィルター**: 直前のSystem1シグナルが利益だった場合はスキップ
  （ただしSystem2の55日ブレイクは無条件エントリーで取りこぼしを防ぐ）

原典は**先物の多市場分散**（穀物・金利・通貨・株価指数など無相関な
複数市場を同時にトレード）が前提で、単一市場の結果だけでは評価できない
設計であることが重要。

### 現代データへの実装（`turtle_trading.py`）

上記ルールを、日本上場個別株・ロングオンリー向けに忠実に実装した
（True Range計算・N・Donchianチャネル・ユニットサイジング・2Nストップ・
ピラミッディング・whipsawフィルターをすべて含む）。

### 実測結果（10銘柄・5年・単一市場）

| 銘柄 | タートルズ最終資産 | B&H最終資産 | 取引数 | 勝率 |
|---|---|---|---|---|
| 7203.T | 789万円 | 1,722万円 | 21 | 14% |
| 8306.T | 4,068万円 | 7,561万円 | 12 | 50% |
| 9433.T | 1,088万円 | 1,938万円 | 20 | 15% |
| ... | ... | ... | ... | ... |

**10銘柄すべてでB&Hに劣後（0/10）。** 勝率も11〜50%と低め。これは
「タートルズが機能しない」ことの証明ではなく、**単一の持続的な上昇
トレンド銘柄では、頻繁な損切り（トレンドフォローの本質）がB&Hに
勝てない**という、この検証設計の限界そのものを示す結果。原典の実際の
価値は、暴落局面での損失限定と、多数の無相関市場を同時に走らせることで
得られる分散効果にあり、単一株での再現にはそもそも無理がある。

## 2. ウォーレン・バフェットのバリュー投資

### 要約（出典付き）

Frazzini, Kabiller & Pedersen (2018) "Buffett's Alpha"（AQR Capital
Management、*Financial Analysts Journal* Vol.74 No.4）は、バークシャー・
ハサウェイの長期リターン（Sharpe比0.79）を伝統的リスクファクターに
回帰分解した学術研究。結論:

- バフェットの「アルファ」は、**Betting-Against-Beta（安全性）**と
  **Quality-Minus-Junk（クオリティ）**ファクターをコントロールすると
  統計的に有意でなくなる
- バフェットは平均して**約1.7倍のレバレッジ**をかけていたと推定される
- 結論として、バフェットのリターンは「魔法」ではなく、**割安・安全・
  高クオリティな株に適度なレバレッジをかけ続けたこと**として説明できる

### 現代データへの実装（`buffett_style.py`）

この分解をそのまま実装した:

- **Quality** = ROE + 粗利率（Yahoo財務データ）
- **Value** = 1/PER + 1/PBR（同上）
- **Safety** = 対ベンチマークの低ベータ（価格ベース・全期間先読みなく計算可能）

3つの合成スコア上位銘柄を選び、レバレッジ有無での違いを検証する。

> ⚠️ Quality/Valueは**現在のスナップショット**にしか依拠できないため、
> 過去バックテストへの一律適用は先読みバイアスになる（`aqm-strategy/
> quality.py`等と同じ制約）。本検証は「今日この基準で選んだ銘柄群を、
> 過去5年間保有していたら」という条件付きの参考値であり、動的な銘柄
> 入替を伴う正式なバックテストではない。

### 実測結果（10銘柄ユニバースから上位5銘柄選定・5年）

選定銘柄は通信株（NTT・KDDI）中心——低ベータ(0.07〜0.08)・高い
Safetyスコアで、直感的にも「安全株」という選定と整合する。

| レバレッジ | 最終倍率 | 最大DD |
|---|---|---|
| 1.0倍 | 1.96倍 | -20.2% |
| 1.5倍 | 2.56倍 | -29.2% |
| 1.7倍（AQR推定値） | 2.82倍 | -32.5% |
| （参考）ユニバース全10銘柄・均等保有 | **3.28倍** | - |

**レバレッジ1.7倍でもユニバース平均に見劣りする結果になった。** これは
恣意的な結果ではなく、**この5年間が強気相場で、防御的な低ベータ銘柄
（通信株）が相対的に出遅れた**という、市場環境依存の正直な結果。
AQR論文も、バフェットの真の優位性は数十年という長期・複数の景気循環を
通じた頑健性にあると指摘しており、5年・10銘柄という小規模な検証では
再現しきれない性質のものである。

## 3. Joel Greenblatt の Magic Formula

### 要約（出典付き）

*The Little Book That Beats the Market* (2005)。「割安（Earnings Yield
= EBIT/Enterprise Value）」と「高品質（Return on Capital = EBIT/
(正味運転資本+純固定資産)）」の2指標だけで銘柄を順位付けし、両方の
順位合計が良い銘柄を買って1年保有・入替する。原典は**金融株・公益株を
対象外**とする（資本構成の性質が異なりEV/ROCの意味が崩れるため）。

### 現代データへの実装（`magic_formula.py`）

YahooはEBIT・正味運転資本・純固定資産を直接提供しないため、以下で近似する:
`EBIT_proxy = 売上高×営業利益率`、`投下資本_proxy = Enterprise Value − 現金`。
**精密な原典の値ではない近似**であることをコード内に明記している。

### 実測結果と発見（`pipeline.py`の共通⑥〜⑧を適用）

- **⑨⑩の実証**: 10銘柄ユニバースに銀行株(8306.T)を含めて計算したところ、
  **Enterprise Valueが負値**になった。原典が金融株を除外する理由が、
  まさにこの実データで裏付けられた
- 除外ルールを外して計算すると、銀行株のEarnings Yieldは`-0.414`という
  無意味な値になったが、**今回はcombined_rankが10銘柄中最下位に沈んだため
  上位N銘柄の選定は結果的に汚染されなかった**。ただしこれは偶然であり
  （符号の組み合わせ次第では上位へ紛れ込みうる）、除外ルール自体が
  不要という意味ではない——という誠実な結論に至った
- ⑦感度分析（N=3/5/8）・⑧レジーム分析（強気年率+45.0% vs 弱気年率-43.8%）
  も実施

## シリーズ共通の考察（3手法から）

- **何が本質だったか**: いずれも「予測しない・ルールに従う・分散/長期で
  優位性が発揮される」という規律の設計。個別の数式より規律の徹底が本質
- **現代では何を変えるべきか**: 点in-time財務データ（J-Quants等）への
  移行、金融/公益株など前提が崩れるセクターの自動検出、複数資産クラスへの拡張
- **AIだから改善できる部分**: 大量データの解析・複数戦略の並行検証・
  セクター例外や異常値（EV<0等）の自動検知——本シリーズで実際に発見した
  ようなデータ起因の落とし穴を、人手より速く発見できる

## 実装中に見つけたバグ（教訓として記録）

`magic_formula.compute_magic_formula`で当初、「金融セクター除外」と
「異常値（EV非正）チェック」を1つのフラグに束ねていたため、除外ルールを
無効化する比較実験をしても、異常値チェックが結局同じ銘柄を弾いてしまい、
「除外あり/なし」の比較が常に同じ結果になる、というテストの意味が
なくなるバグがあった。両フラグを独立させ、初めて「本当に何の安全策も
なかった場合」を再現できた。

## 限界・注意

- 単一株・小規模ユニバース（10銘柄）・5年間という限定的な検証
- Quality/Value/Earnings Yield/ROCはライブ断面のみ。過去バックテストへの
  適用は先読み
- タートルズは先物の多市場分散が前提だが、本実装は個別株・ロングオンリー
  の簡略版
- Magic FormulaのEBIT/投下資本は近似値。原典の精密な財務諸表項目ではない
- 発注機能なし
