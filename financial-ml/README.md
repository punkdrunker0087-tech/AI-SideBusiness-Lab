# 金融ML研究フレームワーク

「精度の高いモデル」でなく「将来データでも再現するモデル」を目指す金融MLの
研究プロセスを7段階で実装。線形・決定木系・ニューラルネットワークを同一条件
で比較し、**Purge/Embargo付き時系列検証**の効果を実データで定量化する。

> 研究・学習用。発注機能は持たない。

## 研究プロセスと実装の対応

| # | プロセス | 実装 |
|---|---|---|
| 1 | 研究仮説 | `hypothesis.py`（モデル化したい現象・根拠を先に固定） |
| 2 | データ整備 | `data_prep.py`（整合性チェック・欠損/外れ値処理・**将来情報混入の自動検証**） |
| 3 | 特徴量設計 | `features.py`（先読みなしの特徴量＋(date,symbol)長形式データセット構築） |
| 4 | モデル比較 | `models.py`・`evaluation.py`（線形/決定木系/ニューラルネットを同一条件で） |
| 5 | 時系列検証 | `validation.py`（**Purge+Embargo付きwalk-forward**） |
| 6 | ロバスト性評価 | `robustness.py`（強気/弱気レジーム別のOOS性能） |
| 7 | 運用監視 | `monitoring.py`（入力分布ドリフト・特徴量重要度・性能トレンド） |

## セットアップ & 実行

```bash
cd financial-ml
pip install -r requirements.txt
python demo.py
```

## 最重要の実装: Purge / Embargo付き時系列検証

ラベルが「20日先リターン」のように未来の窓を参照する場合、単純な時系列
分割だけでは情報漏洩が起きる。**Purge**は訓練データのうち、そのラベル
期間が検証期間に食い込む分を除去し、**Embargo**はさらに検証直前に
安全マージンを設ける（López de Prado (2018) *Advances in Financial
Machine Learning*）。

### 実測: Purge/Embargoの有無で性能推定がどう変わるか

| モデル | Purge/Embargoあり(IC平均) | なし・単純分割(IC平均) |
|---|---|---|
| 線形(Ridge) | 0.000 | **0.013** |
| 決定木系(RandomForest) | -0.002 | **0.008** |
| ニューラルネット(MLP) | -0.015 | -0.004 |

**3モデルすべてで、単純分割の方がPurge/Embargoありより高いICを示した**。
これは単純分割が情報漏洩により楽観的な性能推定になる、という研究上の
懸念がそのまま実データで確認できた結果。「正しい検証をしないと、実力
以上に見える」という金融MLの核心的な教訓を定量的に示せた。

### 実装中に見つけたバグ（教訓として記録）

当初のEmbargo実装は「直前のtestブロック終了時点」を基準に**訓練データ全体**
をフィルタしていたため、拡大窓（expanding window）の訓練データが
まるごと空になり、4フォールド中3フォールドで学習不能というバグがあった。
正しくは、Embargoは「検証期間の直前」にのみ追加の安全マージンとして
適用すべきで、過去の訓練データ全体を検証ブロックの終了時点基準で
フィルタしてはいけない。`cutoff = test_start - (horizon + embargo)日`
という単一の基準に修正し、4フォールドすべてで正しく学習データが
残ることを確認した。**「動いた（エラーが出ない）」ことと「正しい」
ことは別**、という時系列検証実装そのものの教訓。

## モデル比較で観察されたこと

- **複雑なモデルほど良いわけではない**: 最もシンプルな線形モデル(Ridge)が
  IC=0.000で最良、ニューラルネット(MLP)が最も悪い(-0.015)という結果。
  25銘柄・5年という小規模なデータでは、複雑なモデルは過学習しやすい
- **解釈可能性の違い**: 線形は係数で直接解釈可能、決定木系は特徴量重要度
  （momentum_120が0.29-0.50と一貫して最重要）、MLPは事後説明手法
  （SHAP等）なしには解釈困難

## ロバスト性評価（レジーム別）で見えたこと

強気/弱気レジーム別にOOS性能を分けると、**弱気相場ですべてのモデルの
ICが悪化**（MLP: -0.005→-0.113、RandomForest: 0.004→-0.022）。
全期間平均のICだけを見ていると、この局面依存性は見えない。

## 運用監視（Stage 7）

- **入力分布ドリフト(PSI)**: 全特徴量が0.1未満で「○安定」。ただし
  これは検証に使った5年間内の前後半比較であり、将来の市場構造変化を
  保証するものではない
- **特徴量重要度の推移**: momentum_120がフォールドを通じて一貫して
  最重要（0.29-0.50）。重要度の順位が大きく入れ替わらないことは、
  少なくともこの期間内では特徴量の説明力が安定していたことを示唆する

## 限界・注意

- 25銘柄・5年という小規模データでの検証。IC水準は総じて小さくゼロ近傍
- 生存者バイアス未対応（無料データの制約）
- ラベルは20日先の**クロスセクション順位**のみ。回帰精度そのものは評価対象外
- Embargoの日数・Purgeの正確な期間設定は簡略化されている
- 発注機能なし
