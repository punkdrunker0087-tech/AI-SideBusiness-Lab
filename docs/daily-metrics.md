# daily-metrics.md — 日次実績ログ

AIエージェントはX・BOOTH・GitHubの実数値を自動取得できない（API連携
なし）。このログは人間（CEO）が報告した数字だけを記録する、正直な
実績台帳。`/morning-brief`はここの直近2件を比較して前日比を出す。

未報告の日は空欄のままにし、architecture上は「未報告」として扱う
（0で埋めて実績があったかのように見せない）。

## 日次報告テンプレート（2026-07-14改訂・簡略化）

日次チェックが肥大化した（note記事が増えるたびに記事別ビュー確認が
積み重なった）ため、毎日確認する項目を最小限に絞る。詳細は週次に回す
（`knowledge/Decision Log/2026-07-14-daily-report-simplification.md`）。

**毎日（30秒目標）**:
- 無料版DL合計（台帳＋レジ）
- 製品版売上（台帳＋レジ）
- Xフォロワー数

**週次のみ（CEO定例レビューの回にまとめて）**:
- **BOOTHアクセス数（各商品ページの流入実数）** ← 2026-07-20追加。
  戦略判断の前提条件（`knowledge/Decision Log/2026-07-20-access-
  measurement-precondition.md`）。views≈0=reach問題／views>0＆DL=0=
  offer問題で打ち手が正反対。大きな判断の前は週次を待たず随時取得する
- note/Zennの記事別ビュー・スキ・コメント
- X投稿の週間本数
- BOOTHお気に入り・GitHubスター（変化があれば随時でも可）

CEO Scoreの計算式（Marketing.md）に必要なのは「今週のnote投稿*本数*」
だけで記事別の内訳は不要なため、この簡略化はスコア計算に影響しない。
