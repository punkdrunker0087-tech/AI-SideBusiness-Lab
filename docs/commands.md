# commands.md — PMコマンド仕様

このドキュメントは、AI-SideBusiness-Lab（会社OS）の「ユーザーインターフェース」
であるコマンド群の仕様を定義する。人間（CEO）が直接触るのはエージェントでは
なくコマンドであり、コマンドの裏側でCEO.md／PM.md／各部門契約が呼び出される。

```
CEO
  ↓
PM
  ↓
Commands（UI）
  ↓
Departments（Research / Evaluation ...）
```

## データストア

すべてのコマンドは `docs/portfolio-dashboard.md` を唯一の真実の源
（single source of truth）として読み書きする。案件の新規登録・状態更新・
優先度づけは、必ずこのファイルへの反映をもって完了とする。

## 自然言語インターフェースの前提

各コマンドはスラッシュコマンド（例: `/next`）だけでなく、同じ意図を持つ
自然言語（例: 「今日は何をすればいい？」）からも同一ロジックを呼び出せる
ことを前提に設計する。スラッシュか自然文かに関わらず、実行する処理・
参照する契約・出力形式は共通でなければならない。

---

## /next

| 項目 | 内容 |
|---|---|
| 目的 | 「今考えなくていい」状態を作る。今すぐ着手すべきタスクだけを提示する |
| 入力 | なし（引数不要） |
| 処理 | Portfolio Dashboardを走査し、各案件についてPM.mdのRouting Rulesを適用してExit Criteria達成/未達を判定。次アクションが発生する案件と、Weekly Review未実施などのPM側タスクを抽出する |
| 出力 | 優先度順の「今やるべきこと」リスト（案件名・現在の部門・次のアクション） |
| 呼び出す契約 | PM.md（Routing Rules, Portfolio Prioritization） |
| 呼び出すエージェント | PM |
| 異常時の挙動 | Portfolio Dashboardが空、または全案件が待機状態の場合は「現在アクションが必要な案件はありません」と表示する |
| 自然言語例 | 「今日は何をすればいい？」「次に何をやればいい？」 |

## /pm

| 項目 | 内容 |
|---|---|
| 目的 | 会社全体の状況を一望する「ホーム画面」を表示する |
| 入力 | なし |
| 処理 | Portfolio Dashboardを集計し、Today's Summary（新規・開発中・収益化・改善中の件数）、Today's Recommendation（Routing Rulesに基づく推奨アクション上位3件）、Priority（優先度順の案件）、CEOへの報告事項（エスカレーション対象）を生成する |
| 出力 | Today's Summary / Today's Recommendation / Priority / CEOへの報告事項の4ブロック |
| 呼び出す契約 | PM.md（Weekly Review・Portfolio Prioritization・Escalation Rulesと同じ集計ロジックを日次で適用） |
| 呼び出すエージェント | PM |
| 異常時の挙動 | Dashboardにデータがない場合は「登録済みの案件がありません。/intake から開始してください」と案内する |
| 自然言語例 | 「会社の状況を教えて」「今どうなってる？」 |

## /intake

| 項目 | 内容 |
|---|---|
| 目的 | 新規アイデアを会社に受け付ける唯一の入口 |
| 入力 | 自由記述のアイデア（例: 「YouTube動画要約サービス」） |
| 処理 | 新規ID（PJ-XXX形式の連番）を採番し、Portfolio Dashboardへ状態「新規」で登録した上で、市場調査部への調査依頼（状態「調査中」）に更新する |
| 出力 | 採番されたID、登録内容の確認、市場調査部への引き渡し完了メッセージ |
| 呼び出す契約 | docs/department-contracts.md（市場調査部のInputs/Exit Criteria） |
| 呼び出すエージェント | PM → 市場調査部 |
| 異常時の挙動 | 入力が空、または既存案件と明らかに重複する場合は登録前に確認を求める |
| 自然言語例 | 「新しい案件を登録したい」「こういうアイデアがあるんだけど」 |

## /route

| 項目 | 内容 |
|---|---|
| 目的 | ある案件の現在の状態を確認し、Exit Criteriaに基づき次工程へ進める |
| 入力 | 案件ID（例: `/route 021`） |
| 処理 | 指定案件の現在の担当部門・成果物を確認し、PM.mdのRouting Rulesに照らしてExit Criteria達成/未達・スコアを判定。前進・差し戻し・廃棄・CEOエスカレーションのいずれかを実行し、Portfolio Dashboardを更新する |
| 出力 | 現在の部門・判定材料（Signal数やスコアなど）・次の行き先 |
| 呼び出す契約 | PM.md（Routing Rules）、docs/decision-framework.md、docs/department-contracts.md（該当部門のExit Criteria） |
| 呼び出すエージェント | PM |
| 異常時の挙動 | 指定IDが存在しない場合はエラーを返す。Routing Rulesに該当ルールがない状態の場合はCEOへエスカレーションする |
| 自然言語例 | 「021の状態を進めて」「案件021は次どこに行く？」 |

## /portfolio

| 項目 | 内容 |
|---|---|
| 目的 | 全案件を利益・優先度・状態で一覧化する、CEO向けの俯瞰ビュー |
| 入力 | なし |
| 処理 | Portfolio Dashboard全体を、利益（実績/見込み）順にランキングし、改善・撤退候補・新規の件数を集計する |
| 出力 | 案件数、利益ランキング（優先度スター付き）、改善件数、撤退候補件数、新規件数 |
| 呼び出す契約 | PM.md（Portfolio Prioritization） |
| 呼び出すエージェント | PM（CEO専用ビュー） |
| 異常時の挙動 | 案件がない場合は空のポートフォリオである旨を表示する |
| 自然言語例 | 「全案件を見せて」「一番伸びている事業は？」 |

## /review

| 項目 | 内容 |
|---|---|
| 目的 | 週次レビューを自動生成し、CEOが5分で経営判断できる状態にする |
| 入力 | なし（対象期間は直近1週間がデフォルト） |
| 処理 | 直近1週間のPortfolio Dashboardの変化を集計し、開始件数・Go/No-Go件数・利益変化・改善KPI・停滞案件を抽出する |
| 出力 | PM.mdのWeekly Review構成（利益ランキング・停滞案件・期限切れ・Go案件・No-Go案件・CEOへの報告）に沿ったレポート |
| 呼び出す契約 | PM.md（Weekly Review） |
| 呼び出すエージェント | PM |
| 異常時の挙動 | 直近1週間に変化がない場合は「今週の変化はありません」と明記した上でレポートを生成する |
| 自然言語例 | 「今週のレビューをまとめて」「週報を作って」 |

## /status

| 項目 | 内容 |
|---|---|
| 目的 | 個別案件の現在状態を確認する |
| 入力 | 案件ID |
| 処理 | Portfolio Dashboardから該当案件の状態・担当部門・KPI・履歴を取得する |
| 出力 | 案件ID・案件名・現在の状態・担当部門・主要KPI・直近の更新内容 |
| 呼び出す契約 | docs/department-contracts.md（該当部門のKPI定義） |
| 呼び出すエージェント | PM |
| 異常時の挙動 | 指定IDが存在しない場合はエラーを返し、近いIDや案件名を候補として提示する |
| 自然言語例 | 「021は今どうなってる？」 |

## /search

| 項目 | 内容 |
|---|---|
| 目的 | 案件を名称・キーワード・状態などで検索する |
| 入力 | 検索キーワード（案件名の一部、状態、担当部門など） |
| 処理 | Portfolio Dashboardをキーワードでフィルタし、該当する案件一覧を返す |
| 出力 | 該当案件のID・案件名・状態の一覧 |
| 呼び出す契約 | なし（Dashboard参照のみ） |
| 呼び出すエージェント | PM |
| 異常時の挙動 | 該当件数が0の場合は「該当する案件が見つかりません」と表示する |
| 自然言語例 | 「動画関連の案件を探して」「撤退候補を教えて」 |

## /help

| 項目 | 内容 |
|---|---|
| 目的 | 利用可能なコマンド一覧を役割別に表示する |
| 入力 | なし |
| 処理 | コマンド一覧を CEO向け／PM向け／Departments向け／System に分類して整形する |
| 出力 | カテゴリ別のコマンド一覧（本ドキュメントの目次に相当） |
| 呼び出す契約 | なし |
| 呼び出すエージェント | System |
| 異常時の挙動 | なし |
| 自然言語例 | 「何ができるの？」「コマンド一覧を見せて」 |

## /ceo

| 項目 | 内容 |
|---|---|
| 目的 | CEOに経営相談を持ちかけ、decision-framework.mdに基づく判断・方針を得る |
| 入力 | 自由記述の質問（例: 「今月利益を上げるなら何を優先すべき？」） |
| 処理 | Portfolio Dashboardと直近のWeekly Reviewを踏まえ、CEO.mdの意思決定フロー・decision-framework.mdの基準に従って回答する |
| 出力 | 対象案件・判断・根拠数値・PMへの指示、というCEO.mdの出力形式に沿った回答 |
| 呼び出す契約 | CEO.md、docs/decision-framework.md |
| 呼び出すエージェント | CEO |
| 異常時の挙動 | 判断に必要なデータ（Dashboard・KPI）が不足している場合は、分析部への調査依頼が必要な旨を回答する |
| 自然言語例 | 「今月利益を上げるなら何を優先すべき？」「止まっている案件はある？」（後者は/nextやPM相談に近い場合、PMへ委譲してもよい） |
