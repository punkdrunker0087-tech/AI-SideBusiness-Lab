# portfolio-strategy.md — Portfolio運用ルール

このドキュメントは、会社全体のポートフォリオ運用ルールを定義する。
個別案件の評価ロジック（8軸スコアリング）は `docs/decision-framework.md`
を正とし、本ドキュメントは「評価結果をどう会社運営に反映するか」という
運用ルールを正とする。CEOはこのルールに従ってポートフォリオを管理し、
案件ごとにブレた判断をしない。

---

## 同時進行数の上限

**MAX Active Projects: 3**

案件数が増えるほど、CEO・PMが1件ごとにかけられる意思決定の質は下がる。
10件を同時に走らせるとAIでも管理不能になるため、アクティブ枠を3件に
制限する。

- 4件目以降の新規案件は、Backlog（新規・着手待ち）として保持する
- アクティブ枠に空きができ次第、Backlogの中から優先度の高い案件を
  選んで着手する
- 案件の種類は意図的に分散させる（例: B2C×B2B、競争激烈×ニッチ、
  サブスク型×アフィリエイト型）ことで、会社OSが多様な案件を正しく
  さばけるかを継続的に検証する

## Portfolio Rule（判定 → アクション）

`docs/decision-framework.md` の5段階判定を受け、以下のアクションに従う。

| 判定 | アクション |
|---|---|
| Strong Go | 即着手（アクティブ枠に空きがあれば即座に着手。枠がなければ最優先でBacklog先頭へ） |
| Go | 着手候補（アクティブ枠が空き次第、優先的に着手） |
| Conditional Go | 保留。**2週間以内**に改善案（ニッチ特化・追加検証結果など）を提出しなければ、自動的にPivot扱いとする |
| Pivot | 改善。事業評価部・市場調査部と方向転換案を協議し、再評価する |
| No-Go | 終了。**30日間**アーカイブとして保存した後、Portfolio Dashboardの案件一覧から除外する |

## 収益ルール

- 運用中案件の利益率が **20%未満** の場合、改善対象とする（改善部へ依頼）
- **3ヶ月連続で利益ゼロ**の案件は撤退候補とする
  （`docs/decision-framework.md`「2. 運用中案件の継続／改善／撤退判定」と連動）

## アクティブ枠の管理

- PMはPortfolio Dashboardのアクティブ案件数を常に把握し、上限（3件）を
  超えないよう管理する。Portfolio Dashboard冒頭にアクティブ案件数を
  「アクティブ案件数: X / 3」の形式で明示する
- アクティブ枠を超えて着手してはならない。超過が発生しそうな場合は
  PMがCEOへエスカレーションする
