# ai-native-pipelines.md — AIが最適稼働できるパイプライン候補（10）

作成: 2026-07-13（CEO依頼）。「AIが最適に稼働できるパイプライン」を
実測ベースの判定軸で10個列挙する。個々を着手する前には必ず
`knowledge/SOP/demand-discovery.md`（10件バッチ・5問スクリーニング）で
需要検証する——このリストは"起点"であって承認ではない。

## 判定軸（この会社の実測から）
1. **AIが価値の中核を担える**: 出力が「文章・コード・構造・合成」で
   完結する。人間の職人判断（PJ-008の"面白い部分への編集"）に依存しない
2. **Build Once, Sell Many**: 一度作れば売り続けられる（PJ-005の労働
   集約=不適合）
3. **需要検証可能**: 既存の有料商品・レビューで支払いの証拠を取れる
4. **🔴を踏まない**: 認証情報の保持・本番公開・対外送信・金銭授受を
   AI単独でやらない設計にできる
5. **週末規模・低コスト**: CEOの可処分時間を圧迫しない

各行の「AI比率」は、企画→制作→QA→パッケージのうちAIが単独で担える
割合の体感値。人間は原則「公開ボタン」だけ。

## 選定ロジックと優先度（正直な見立て）

- **1・2・3（Tier 1）は"作れば売れる状態"に最も近い**。ただし——
- **今の会社の律速は「作る力」ではなく「届ける力」**（売上・DL 5日連続0、
  認知/リーチ不足と実測で確定）。だから production 系を増やすより、
  **#10（買い手向けコンテンツ・エンジン）が戦略的に最も価値が高い**。
  作った9商品が売れていない状態で11個目を作るのは「作るへの逃避」
  （`docs/pre-mortem-90days.md`シナリオ5）
- したがって推奨順は **#10 →（既存商品の販売が動いてから）#1/#4/#5**
- #6は著作権、#8は文化差、#9は出典管理が地雷。着手時に工程0で潰す

## 共通の非対象（この会社では作らない）
- 代行・受託（PJ-005/008で確認: 労働集約・職人判断依存・堀なし）
- AIが認証情報を持って自動ログイン・自動投稿・自動出品する類（🔴）
- 実装ごとに作り直しが必要で資産にならないもの（原則8違反）

# 追補（2026-07-13）: 「AIが24時間自律稼働して稼ぐ」観点での再提案

CEOの再定義: パイプラインの理想は「AI社員が24時間自動稼働して"稼ぐ"」。
この軸で上のリストを組み替える。

## 先に確定すべき制約（正直に）
- **完全無人の"稼ぐ"は現行ルールと両立しない**。稼ぎのラスト1マイル
  （公開・課金・対外送信・認証情報保持）は全て🔴＝AI単独禁止。
  憲法「意思決定は人間に帰属」＋各プラットフォーム規約（自動出品・
  自動投稿はアカウント凍結対象）に直結する
- **達成可能な最良形**: 「AIが24時間ループを回し続け、人間はゲートで
  まとめて数秒承認するだけ。公開済み資産はその後24時間受動的に売れる」。
  稼ぎのイベント（購入）は自動で24/7発生する。人間の関与を"連続作業"
  から"低頻度のバッチ承認"へ最小化するのが正しい目標
- したがって評価軸は「**AIループの連続性 × 稼ぎへの近さ × 人間ゲートの
  少なさ**」。以下はその順にランク

## Tier A — 最も24/7自律稼働に近い（AI連続生成→人間バッチ承認→公開後は受動課金）

1. **在庫拡張ファクトリー・ループ**: AIが需要検証→制作→12項目QA→
   パッケージを**連続実行**し「公開待ちキュー」に積み続ける。人間は
   週次でまとめて公開承認。公開済み資産はBOOTH等で24時間受動販売。
   → AIの稼働率が最も高く、稼ぎは文字通り24/7。既存SOPで即始められる
2. **検索資産（SEO）量産エンジン**: 検索意図に沿う実用記事・比較・
   早見表をAIが連続生成→（承認後）公開。検索流入は公開後ずっと
   積み上がり24/7でトラフィック→商品導線/アフィリで収益。AIの
   synthesisが最も効き、**現在の律速"届ける"に直接効く**
3. **買い手向けコンテンツ・エンジン（SNS/note）**: note-005型を連続
   量産。人間は🟡承認。継続発信でリーチを24/7じわじわ拡大

## Tier B — 自律ループだが人間ゲート頻度or収益が間接

4. **セルフ最適化ループ（OODA自動化）**: 日次で数字を観測→出品文・
   価格・サムネの改善案を生成→人間承認→反映。既存の稼ぎ面を連続改善。
   `/morning-brief`＋21:00リマインダーの発展形で、**最も安全に自律度を
   上げられる現実路線**
5. **アップデート型データ商品**: 「常に最新」が価値の商品（相場・
   ランキング・規則まとめ）をAIが定期更新。継続購入/サブスクの理由に。
   人間はリリース承認
6. **多言語横展開ループ**: 既存の売れ筋を各言語へ連続ローカライズし
   別市場に供給。翻訳＝AI最適、限界費用ほぼ0で"在庫の面積"を24/7拡大

## Tier C — 自律性は高いが要注意（コスト/規約/著作権/信頼リスク）

7. **アフィリエイト型比較コンテンツ**: AIが比較記事を量産しリンク収益。
   24/7で稼ぐが、低質量産はブランド毀損＋検索ペナルティ＋規約リスク。
   憲法「感動品質」と衝突しやすく慎重運用が必須
8. **デジタル素材のオンデマンド供給**: AIがテンプレ・配色・アイコン
   セット等を連続生成しストックに積む。人間は公開承認。独自性・
   著作権の担保が前提
9. **無料診断→有料導線のリード獲得ツール**: 24/7で見込み客を拾い商品へ。
   ただしスコアリング系は過去に需要実証不能で見送った型＝需要検証必須
10. **育つナレッジ商品**: 一分野を深掘りし続け、章を足すたび価値と
    価格が上がる。AIの連続執筆が資産複利になる（人間は増補リリース承認）

## 正直な結論（この軸での最適解）
- **稼ぎを24/7化する最大のテコは、production種の追加ではない**。棚
  （9商品）は既にあるのに流入0。**受動的24/7課金には24/7の"発見"が
  要る**ので、AIが連続・安全に回せる #2/#3（検索・買い手コンテンツ）が
  本命。#1（在庫ループ）はその発見が効き始めてから増産で乗せる
- **今すぐ自律度を安全に上げるなら #4**。稼ぎ面の監視・改善案生成・
  下書き・キューイングは🟢〜🟡で回せ、既に一部（brief・リマインダー・
  週次レビュー自動化）が稼働中。ここを厚くするのが"AI社員の24時間化"の
  現実的な第一歩
- 変わらぬ天井: 金銭・公開・対外送信・認証の🔴は人間ゲートに残す。
  「AIが売る」ではなく「**AIが作り・広め続け、人間が承認し、資産が
  24時間売れる**」がこの会社の自律稼働の正しい形
