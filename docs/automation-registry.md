# automation-registry.md — 稼働中の自動化レジストリ

この会社で動いている自動化（Routine・連携）の唯一の一覧。増やしすぎ
防止（Core Rule 06 Automation is Expansion）と、暴走時のキルスイッチの
ため、**新しい自動化を足したら必ずここに追記する**。

## 自動化の大原則（2026-07-14 CEO方針「承認不要はどんどん自動化」）
- **自動化してよい**: 材料の生成・記録・分析・下書き（🟢）
- **人間ゲートに残す**: 公開・投稿・出品・価格変更・Go/No-Go・撤退などの
  **経営判断と対外アクション**（🟡/🔴。憲法「意思決定は人間に帰属」）
- 「承認不要」とは"判断の材料づくり"を指し、"判断そのもの"ではない

## その他の自動化（Routine以外）
- **Zenn自動公開**: `limit-lab-articles`のGitHub連携。`published: true`で
  push→自動公開（フラグ変更＝人間の承認ゲート）
- **週次のdoc/不要Module監査**: 週次レビュー（PM.md 6項目のアーキテクチャ
  健全性）に内包済み。独立Routineは作らない（Reuse First）

## キルスイッチ
CEOが「◯◯の自動化を止めて」と言えば該当Routineを`delete_trigger`で削除
する。全停止は上表の全IDを削除。
