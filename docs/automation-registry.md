# automation-registry.md — 稼働中の自動化レジストリ

この会社で動いている自動化（Routine・連携）の唯一の一覧。増やしすぎ
防止（Core Rule 06 Automation is Expansion）と、暴走時のキルスイッチの
ため、**新しい自動化を足したら必ずここに追記する**。

## 運営ドクトリン（2026-07-14 CEO確定・`CLAUDE.md`第16章）
1. **自動通知で失念を防ぐ**（日次・週次・判定日の自動発火）
2. **最終承認は人間（不変）**: 経営判断（Go/No-Go・撤退・価格）と対外
   アクション（公開・投稿・出品・金銭・認証）は絶対に自動化しない
3. **その他はすべて自動化**: 材料の生成・記録・分析・下書き・判定の計算
- 「承認不要」とは"判断の材料づくり"であって"判断そのもの"ではない
  （詳細: `knowledge/Decision Log/2026-07-14-operating-doctrine.md`）
