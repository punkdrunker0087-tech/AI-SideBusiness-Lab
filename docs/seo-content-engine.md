# seo-content-engine.md — 検索資産(SEO)量産エンジン 設計図

起点: 2026-07-13、CEOが`docs/ai-native-pipelines.md`の総合1位「検索資産
量産エンジン」への着手を決定。本ファイルはその**ループ設計図**（working-
method工程5）と勝利条件。ニッチ/プラットフォームはCEO決定待ち（末尾）。

## これは何を狙うエンジンか
AIが検索意図に沿う実用コンテンツを連続生成→人間がバッチ承認→公開後は
検索流入が積み上がり24時間トラフィックを生む。そのトラフィックを
**自社商品導線＋アフィリエイト**で二重回収する。会社の律速（流入0・
認知不足）に直接効く唯一の連続稼働エンジン。

## 大前提: 売上は「検索需要」で決まる（フォロワー数ではない）
（2026-07-14学び。`knowledge/Decision Log/2026-07-14-search-demand-over-
followers.md`）
- 販売のリーチはフォロワー数ではなく**検索需要との一致**で決まる。
  note/Zennは検索・Google上位・おすすめ露出があり、フォロワー以外が
  記事に辿り着くルートが完成している。**フォロワー78・横ばいは売上の
  ブロッカーではない**——フォロワー増加施策に時間を使いすぎない
- 購入の引き金は「ファン度」でなく「今すぐ悩みを解決したい」。
  勝負の**8割はテーマ選び（＝検索需要の発見）**。書く前の準備に最も
  時間を使う（工程0〜2）
- ただし「テーマが合えば簡単に売れる」は誇張（当社はnote6本＋商品9点で
  売上0が反例）。テーマ適合は**必要条件だが十分条件ではない**。
  インデックス・実際の刺さりには時間と検証が要る

## 勝利条件（生成前に固定・後から動かさない）

⚠️ **SEOは即効性ゼロ。** 新規コンテンツが順位を持つのは数週間〜数ヶ月
先。以下の時間軸で判定し、序盤を売上で判定しない。

| 期間 | 見る指標（先行→遅行） | 合格ライン |
|---|---|---|
| 〜30日 | 公開本数・品質・インデックス | 品質基準を満たす記事をN本公開しGoogle等にインデックス、初インプレッション観測 |
| 〜60日 | インプレッション・クリックの伸び | 検索インプレッションが右肩上がり、記事から自社商品/アフィリへの初クリック |
| 〜90日 | 流入→回収 | 月間流入が積み上がり、商品DL/売上またはアフィリ収益の初計上 |

- 成功の定義=**複利で増えるトラフィック資産**の形成であって、初月の
  売上ではない
- 地雷: 検索スパム（中身の薄い量産）／誇張／アフィリ非明示（憲法の
  信頼と信用に反する。順位ペナルティにもなる）

## コンテンツを2層に分ける（2026-07-15・「AIが書いた記事は3スキ」への対応）
AIが"書いた"きれいな文章は読者に数秒で見抜かれ刺さらない。だから
コンテンツを2層に分け、自動化の対象を間違えない。
- **コモディティ層（AIが著者でよい）**: SEOのhow-to・事実・早見表・
  構造化された実用情報。我々が文書化した一次情報（実測・実体験の事実）を
  素材に、AIが下書きしてよい。自動生成Routineの主対象
- **差別化層＝刺さる層（AIは編集者・人間が著者）**: プロセスエコノミー
  （挑戦の途中経過・生々しい失敗・感情）。**体験と感情はCEOが書き、
  AIは編集・壁打ち・校正に徹する**（PJ-008「作業は自動化・価値の中核は
  人間」を"書く"に適用）。自動Routineが作れるのは骨組み＋声を注入する
  ための問い（下書き）まで。完成品として公開しない
- **我々の最大の差別化コンテンツ＝「売上0の正直な記録」**。Decision Log・
  daily-metrics・撤退（PJ-005/008）・3回の判定は、生々しい一次情報の
  宝の山。整った説明でなく、CEOの実際の声で失敗ごとさらけ出す
- AIの使い方の原則: 「書いて」でなく「読者目線で弱い部分を指摘して／
  この体験はどの切り口で役立つ？／誤字と表記ゆれを直して」

## ループ設計図（1記事=1サイクル）

| 工程 | 担当/リスク | 内容 |
|---|---|---|
| 0 収益導線の逆算 | 🟢AI | 「書きたいこと」でなく「読ませたら自社の稼ぎに繋がるか」からテーマを選ぶ |
| 1 キーワード選定 | 🟢AI | **ミドル〜ロングテール（月間検索100〜1,000目安）**の実用クエリを1つ。超ビッグワードはゼロ権威では勝てない。飽和チェック（実検索）必須 |
| 2 検索意図の一行言語化 | 🟢AI | 上位5件を見て「読者は何を解決したいのか」を1行にする。これを構成・本文の軸にする |
| 3 構成 | 🟢AI | 読者の悩み→結論先行。**冒頭100字で「この記事の答え」を言い切る**（GEO対応・下記） |
| 4 本文生成＋一次情報 | 🟢AI | **自社にしかない一次情報を必ず1つ以上入れる**（実測値・運営の実体験・正直な失敗）。E-E-A-Tの"経験"はAIには無い＝ここが唯一の堀。ブランド語りは末尾 |
| 5 破壊者＋記事QA | 🟢AI | 下記「記事QAチェックリスト」を通す。役立つか・薄くないか・誇張・AI臭い定型表現・数値/年号の誤り・検索意図に答えているか |
| 6 収益導線 | 🟡人間承認 | 自社商品／アフィリリンク挿入（**アフィリは明示**・実際に価値を認めるものだけ） |
| 7 公開 | 🟡人間承認 | `published: true`に切替えpush（🔴自動投稿はしない） |
| 8 計測・記録 | 🟢AI | 公開日・URL・狙いKW・後日の反応を`seo-content-log.md`に記録 |
| 9 公開後リライト | 🟢AI→🟡承認 | 数値トリガー（下記）でリライト。新規と並行し、3ヶ月経過記事を月数本 |

### 記事QAチェックリスト（工程5・単一HTMLの12項目QAに相当）
- [ ] 自社の一次情報（実測・体験・失敗）が1つ以上入っている
- [ ] 冒頭2段落が「のっぺりしたAIの書き出し」でなく、読者の痛み/答えから始まる
- [ ] 冒頭100字で記事の答えを言い切っている（GEO）
- [ ] 「〜と言えるでしょう」「〜かもしれません」等のAI臭い定型を削除
- [ ] 数値・年号が最新かつ正確（事実確認）
- [ ] 検索意図（工程2の1行）に本文が正面から答えている
- [ ] 内部/外部リンク・自社商品導線が自然に入っている
- [ ] 誇張・断定回避（税務/法務など）・アフィリ明示

### GEO（生成エンジン最適化）— ゼロ権威の新規に効く近道
AI Overviews/Perplexity/ChatGPT Searchに**引用される**ことを狙う。
検索1位のCTRが大きく落ちている今、ゼロ権威の新規がGoogle SERPで勝つより、
AI回答に引用される方が現実的な露出ルートになりうる。3原則:
1. 冒頭100字で答えを言い切る 2. 一次情報を必ず入れる（AI検索は独自
データを好んで引用）3. 記事全体が「問いへの回答」として機能する構造

- **稼働頻度**: 週2〜3本（継続可能なペース。量より継続がSEOの本質。
  最初の2週間は「型」を固めることに全力＝月30本を最初から狙わない）
- **1サイクル1記事**（同時に多変数を変えず、何が効いたか測れる形に）
- **停止/継続条件**: 90日でインプレッションが全く付かないKW群は捨て、
  付いたトピックに資源集中（撤退を恐れない）

### 公開後リライトの数値トリガー（工程9・SEOは公開後3〜6ヶ月で確定）
| 指標 | リライト不要 | 要観察 | 即リライト |
|---|---|---|---|
| 検索順位 | 1〜10位 | 11〜20位 | 21位以下 |
| インプレッション | 増加傾向 | 横ばい | 減少傾向 |
（Zennでは滞在時間/直帰率の細かい計測は限定的。順位・インプレを主に見る）
リライトは①鮮度更新②検索意図の再調整③一次情報の追加。全文書き直しは
しない（タイトル・冒頭・H2の微調整から）。

## 外部参考の取り込みと、Zennでの適用限界（2026-07-13）
CEO共有の「生成AI×SEO量産」参考記事から上記を取り込んだ。ただし
**破壊者を通し、我々に効くものだけ採用**した。正直な適用限界:
- 参考記事は**自社サイト前提**（Search Console/GA4/FAQ構造化データ/
  直帰率チューニング）。我々は当面**Zenn**で、借り物のドメイン権威で
  順位化の速さを取る代わりに、**技術SEO・スキーマの制御は効きにくい**。
  FAQ構造化データやGA4精密計測が本命の武器になる段階で、自社サイトへの
  拡張を再検討する（Graduation First）
- 有料ツール（Ahrefs/Semrush）は当面不要。無料（Search Console＋Zenn
  アナリティクス＋Ubersuggest無料）で足りる。月1万PV超で再検討
- 我々の強みは既に持っている: **「AIに会社を運営させている」実体験**＝
  E-E-A-Tの"経験"。参考記事が「AIには出せない」と言う一次情報を、
  我々は毎日の運営ログとして生産している。ここに全記事を接地させる

## リスク色分け（working-method）
- 🟢 AI単独可: KW選定・構成・執筆・自己検査・記録・分析
- 🟡 人間承認: 収益リンク挿入・公開・アフィリ提携
- 🔴 禁止: 自動投稿・認証情報の保持・被リンク購入等のブラックハットSEO

## 確定した設定（2026-07-13 CEO決定）
- **ニッチ/読者 = AI活用・個人開発・副業**。理由: 我々のnoteで実際に
  反応がある層で、「AIで会社を運営している」実体験が一次情報として
  強い（真正性）。アフィリ市場が豊富（AIツールのサブスク・ホスティング・
  書籍・講座・高単価）。自社のOS Kit / Prompt Pack / Claude Codeテンプレ
  とも直結し、同じ記事で自社商品＋アフィリを二重回収できる
- **主プラットフォーム = note＋Zenn/Qiita（まずここ。CEO拒否あれば変更）**。
  Zenn/Qiitaは開発者向けで本ニッチと最適合・ドメイン権威が高く順位化が
  速い。自社サイトSEOは資産base ができてから
- **トーン**: 実体験ベースのhow-to。「AIに会社を運営させている実験」から
  得た具体を出す。ブランド語りは末尾（コンテンツスタイル原則）
- **収益導線**: 自社商品（OS Kit / Prompt Pack）＋アフィリ（Claude等の
  AIツール・関連書籍。提携明示・実際に使っているものだけ）

記録・実測は`docs/seo-content-log.md`。
