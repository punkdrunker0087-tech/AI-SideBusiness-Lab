# AIにできること / できないこと（会社設立編）

「君にできないことを調査しながら進める」ための境界線マップ。
結論: **書類のドラフト・調査・比較検討・管理はAIができる。法的効力の発生する行為・本人確認を伴う行為・金銭の移動は全て人間（代表者本人）にしかできない。**

## ✅ AI（Claude）にできること

| 領域 | 具体的にできること |
|---|---|
| 調査 | 制度・費用・税制の最新情報調査、株式会社 vs 合同会社の比較、類似商号の事前チェック方法の案内 |
| 設計 | 事業目的の条文案作成、資本金・役員構成・事業年度のシミュレーション、事業計画書のドラフト |
| 書類ドラフト | 定款の草案、設立時の各種議事録案、税務署への届出書の記入内容案、銀行口座開設の申請文面 |
| LIMIT LAB引き継ぎ | 事業譲渡契約書のドラフト、譲渡資産リストの整理、ブランド移行計画の策定 |
| 設立後の運営 | 会計仕訳の考え方の説明、請求書テンプレート、規程類（就業規則等）のドラフト、このリポジトリでの進捗管理 |
| 開発 | 新会社のWebサイト・LP・サービスそのものの開発（このリポジトリの本来のミッション） |

## ❌ AIにできないこと（＝本人がやることリスト）

| # | 項目 | できない理由 | 誰が/どこで |
|---|---|---|---|
| 1 | 最終的な意思決定（商号・形態・資本金） | 法的責任は発起人に帰属する | 本人 |
| 2 | 電子署名・実印の押印 | マイナンバーカード等による本人の電子署名が必要 | 本人 |
| 3 | 公証役場での定款認証 | 本人（または委任した士業）の出席・ウェブ会議参加が必要 | 本人 / 公証役場 |
| 4 | 資本金の払い込み | 銀行口座の操作・金銭の移動 | 本人 / 銀行 |
| 5 | 登記申請の提出 | 申請者の本人確認・署名が必要 | 本人 / 法務局・ワンストップサービス |
| 6 | 法人印鑑の作成・印鑑届出 | 物理的な作業と届出 | 本人 / はんこ屋・法務局 |
| 7 | 法人銀行口座の開設 | 厳格な本人確認（KYC）と審査 | 本人 / 銀行 |
| 8 | 税務署等への届出の提出 | e-Taxでも本人の電子証明書が必要 | 本人 |
| 9 | 登記・税務の「代理」業務 | 司法書士・税理士の独占業務（AIも無資格者も不可） | 司法書士 / 税理士 |
| 10 | LIMIT LABの譲渡契約の締結 | 契約当事者としての意思表示 | 本人（+相手方） |
| 11 | 法律・税務の個別具体的な助言の確定判断 | 一般的な情報提供はできるが、最終判断は専門家確認が安全 | 弁護士 / 税理士 / 司法書士 |

## 運用ルール

1. AIができる項目は、依頼があれば即ドラフトを作りこのリポジトリにコミットする
2. AIができない項目は、実行前に「持ち物・手順・所要時間・費用」をまとめた実行チェックリストをAIが用意する
3. できる/できないの境界で迷ったら、この表に追記して記録を残す（失敗を記録するというLabの方針に準拠）
