# org-expansion-roadmap.md — 組織拡張構想（未導入）

このドキュメントは、CEOが提示した将来の組織拡張構想を記録するもの
であり、**現時点では導入しない**。`CONSTITUTION.md`の「組織構造との
関係」の通り、実際の事業規模・必要性が実証されてから段階的に導入を
判断する。現在の正式な組織構造は `CLAUDE.md` を正とする
（CEO・PM（COO）・8部門）。

## 構想の全体像

```
CEO
 ↓
役員AI（COO / CTO / CMO / CFO / CAIO）
 ↓
共通本部
・AI戦略本部
・市場調査
・ブランド戦略
・品質保証
・法務・規約確認
・マーケティング
・SNS
・データ分析
・AI研究開発
・システム開発
・ナレッジ管理
 ↓
専門制作部門
・LINEスタンプ
・3Dアバター
・動画制作
・アプリ開発
・AIツール
・Web制作
・電子書籍
・テンプレート
・その他デジタルコンテンツ（拡張可能）
```

## なぜ今は導入しないか

- 現在の会社OSは案件2件（PJ-001・PJ-002）、実働部門8つ、初回売上
  まだ0件の段階にある。役員層5職・共通本部11部門・専門制作部門N個を
  今作っても、実際の業務量に対して組織だけが先行して肥大化する
  （Reality First / No Internal Completionの原則に反する）
- 現行の8部門は、構想の多くを実質的にカバーできている
  - 市場調査部＋分析部 ≒ AI戦略本部（市場・トレンド・ROIの継続分析）
  - マーケティング部 ≒ マーケティング＋SNS（すでに`docs/channel-strategy.md`
    でXを担当）
  - 法務部 ＝ 法務・規約確認
  - 検証部 ≒ 品質保証
  - 制作部の4モード（Software/Content/Template/Package）は、
    専門制作部門（LINEスタンプ・3Dアバター・動画・アプリ等）を
    「モードの追加」として吸収できる可能性が高い

## 段階的導入の目安（トリガー条件）

以下のいずれかに該当した時点で、該当部分の導入を具体的に検討する。

| 構想要素 | 導入を検討するトリガー |
|---|---|
| 専門制作部門（LINEスタンプ・3Dアバター・動画等） | 制作部の4モードでは表現しきれない、明確に異なる制作プロセス・専門知識が必要な事業を実際に着手するとき |
| AI戦略本部（独立部門化） | 市場調査部・分析部の兼務では継続的な機会探索が追いつかなくなるほど案件数・事業数が増えたとき |
| COO/CTO/CMO/CFO/CAIO（役員層） | 部門数が現在の8を大きく超え、PM（COO相当）1人では部門間調整が回らなくなったとき |
| データ分析・AI研究開発の独立部門化 | Sprint 7（分析部）新設条件（販売済み商品1件以上）を満たし、かつ分析業務が定常化したとき |
| ナレッジ管理の独立部門化 | `knowledge/`配下の記録量・案件数が増え、PMの片手間では管理しきれなくなったとき |

## 運用ルール

- この構想を実装する際は、必ずCEOの承認を経て `CLAUDE.md` の組織図・
  `docs/department-contracts.md` を更新する形で行う（このドキュメント
  だけを見て部門を作らない）
- 部門を追加するたびに、既存部門との重複・責務の重なりがないかを
  必ず確認する
