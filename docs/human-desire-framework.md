# human-desire-framework.md — LIMIT LAB Human Desire Framework v1.0

LIMIT LABは、人の欲求を**操作するため**ではなく、**理解するため**に
このフレームワークを用いる。すでに存在する目標・悩み・願いを深く理解し、
その実現を支援する価値を設計し、誠実に伝えることが目的である。
「売るための分類」ではなく、人間理解のフレームワークとして設計する。

この原則は`CONSTITUTION.md`第一条（信頼と信用）・第三条（先義後利）を
実務に落とし込んだものであり、これに反する使い方（誇張・不安の煽り立て・
架空の緊急性の演出等）はBrand Guardianの拒否権の対象となる。

## 運用原則

1. LIMIT LABは、人の欲求を操作するためではなく、理解するためにこの
   フレームワークを用いる
2. 私たちは、人がすでに持っている目標・悩み・願いを深く理解し、その
   実現を支援する価値を設計し、誠実に伝える
3. 短期的な購買よりも、長期的な信頼を優先する

## 6段階の欲求レベル

| Level | 名称 | キーワード | よくある思考 | 刺さる表現例 |
|---|---|---|---|---|
| 1 | 生存・安心（Security） | 安全・安心・時間・お金・健康・リスク回避 | 損したくない／失敗したくない／面倒を減らしたい | 時短・自動化・シンプル・安心設計 |
| 2 | 快適・利便性（Comfort） | 楽・効率・ストレス軽減・整理・快適 | 少ない労力で成果を出したい／面倒な作業をなくしたい | ワンクリック・すぐ使える・初心者でも安心 |
| 3 | 成長・達成（Achievement） | 成果・上達・学習・スキル・生産性 | 成長したい／周囲より成果を出したい | レベルアップ・最短ルート・再現性 |
| 4 | 承認・所属（Recognition） | 評価・信頼・仲間・ブランド・コミュニティ | 認められたい／信頼されたい／仲間が欲しい | 多くの利用者・実績・愛用者の声 |
| 5 | 自己表現（Identity） | 自分らしさ・個性・世界観・デザイン・美意識 | 自分らしくありたい／好きを表現したい | あなたらしく・オリジナル・世界観 |
| 6 | 影響力・貢献（Impact） | 社会・家族・チーム・教育・リーダーシップ | 誰かの役に立ちたい／良い影響を与えたい | 次世代へ・チーム全体・社会に価値を届ける |

## 分析ルール

商品を設計・販売するときは、以下6項目を必ず整理する。

| 項目 | 内容 |
|---|---|
| Primary Desire | 最も強く満たす欲求（Level 1〜6のいずれか） |
| Secondary Desire | 副次的に満たす欲求 |
| Customer Pain | 顧客が抱える課題 |
| Emotional Outcome | 得られる感情 |
| Functional Outcome | 得られる機能的価値 |
| Identity Outcome | ユーザーはどんな人になれるか |

## 誰が・いつ使うか

- **市場調査部**: `Research.md`の「③ Customer」セクションでペルソナを
  記述する際、このフレームワークでPrimary/Secondary Desireを特定する
- **マーケティング部**: `Marketing.md`の販売ページ・商品説明・SEOタイトル
  を作成する前に、この分析ルール6項目を整理してから執筆する。刺さる
  表現例はあくまでヒントであり、誇張・煽りに転じないよう
  `CONSTITUTION.md` Mission Checkと併せて確認する
- **事業評価部**: Value Scoreの「利用者価値」を算定する際の参考情報として
  用いてよい（Primary Desireが明確で、Painに対する解決策が具体的なほど
  利用者価値は高いと判断できる）

## 適用例

### 商品: Claude Code Prompt Pack / AI Company OS Template Kit

| 項目 | 内容 |
|---|---|
| Primary Desire | 成長・達成（Level 3） — AIを使いこなし、成果を出せるようになりたい |
| Secondary Desire | 快適・利便性（Level 2）／安心（Level 1） — ゼロから作る手間を省きたい、失敗したくない |
| Customer Pain | AIを使いこなせない／作業が属人化している／何から始めればいいか分からない |
| Emotional Outcome | 「これなら自分にもできそう」「仕事が前に進む」 |
| Functional Outcome | 開発・検証プロセスの時間短縮、品質の標準化 |
| Identity Outcome | AIを活用して成果を出せる人になる |

### 商品: AI Business Template Pack

| 項目 | 内容 |
|---|---|
| Primary Desire | 快適・利便性（Level 2） — 面倒な書類作成の手間をなくしたい |
| Secondary Desire | 安心（Level 1） — 抜け漏れのない書式で失敗したくない |
| Customer Pain | 議事録・SOP等をゼロから作るのが面倒／書式に自信がない |
| Emotional Outcome | 「これでもう迷わない」 |
| Functional Outcome | 文書作成時間の短縮、抜け漏れの防止 |
| Identity Outcome | 仕事を整理して回せる人になる |
